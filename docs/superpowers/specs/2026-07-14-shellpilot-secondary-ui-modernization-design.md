# ShellPilot 二级界面美化与双语主题设计

**状态：** 已由用户确认

**日期：** 2026-07-14

**目标版本：** 待实施计划确定
**设计范围：** 设置中心、配置界面、应用内右键菜单、UI 主题与简中/英文体验

## 1. 背景与目标

ShellPilot 已具备完整的 SSH、SFTP、终端、AI、同步和主题能力，但设置页、配置表单和应用内右键菜单仍沿用较分散的 Electerm 样式。部分页面层级不足，表单密度与文字长度适配不一致，主题编辑器直接暴露英文变量名，客户端内也存在硬编码或未完整汉化的可见文案。

本项目在不改变业务功能的前提下，建立一套统一的二级界面设计系统。设计重点是清晰、专业、可长期使用，并支持简体中文和 English。主工作台不做布局改造；终端区域在所有主题下保持近黑色。

## 2. 已确认的关键决策

1. 采用“分层卡片式”视觉方向：浅灰页面承托配置卡片，使用明确的标题、说明、边界和状态层级。
2. 采用“视觉统一 + 轻量体验增强”的改造深度。
3. 保留客户端现有 UI 主题能力，改造为主题卡片、即时预览、确认应用和折叠式高级编辑。
4. 保留现有语言能力，重点保证简体中文和 English 完整、无遗漏。
5. 所有主题只影响外围 UI。终端标签区、终端画布和终端空白区域固定使用近黑色 `#0E0F12`。
6. 终端前景色、光标色、选区色和 ANSI 颜色仍可配置，但任何主题都不得覆盖终端背景色。
7. 主工作台布局、工具栏位置、SSH/SFTP/AI 业务逻辑、配置含义和保存语义不在本项目中重构。

主题继续沿用客户端现有的全局 UI 主题行为，因此选择配色后，主工作台已有的顶栏、侧栏等界面可以像现在一样继承主题颜色；本项目不重新布局或重写这些主工作台组件。新增的卡片、表单和菜单规范只应用于本项目范围内的二级界面。

## 3. 范围

### 3.1 包含

- 设置中心全部主分类与子页面：连接、通用设置、UI 主题、快捷命令、配置模板和工具中心。
- 通用设置中的常规、终端、AI 与模型、同步与备份、快捷键和密码管理等页面。
- 新建和编辑连接表单，包括身份验证、代理与跳板、SSH 隧道、启动脚本和高级选项。
- 模型 API 配置、Agent Skill、MCP Server、代理等现有配置内容。
- WebDAV、GitHub Gist、Gitee Gist、自定义服务器等同步与备份配置。
- 应用内基于 Web UI 渲染的右键菜单，包括书签、终端、SFTP、标签页、传输列表和输入框菜单。
- UI 主题选择、主题预览、内置主题、高级颜色编辑、主题导入和导出。
- ShellPilot 可见文案的国际化清理，重点覆盖简体中文和 English。
- 最小窗口、系统缩放和长文案下的响应式布局。

### 3.2 不包含

- 主工作台布局、顶部工具栏、左侧主工具栏、AI 主面板和终端操作流程的重新设计。
- SSH、SFTP、终端、AI、同步、更新或安全中心的业务逻辑变更。
- 配置字段语义、默认值、持久化格式和 IPC 接口的主动变更。
- Electron 原生系统菜单的视觉定制；原生菜单继续由操作系统绘制。
- 删除客户端已有语言。现有语言继续可用，但本项目只承诺简体中文与 English 的完整质量。

## 4. 设计架构

### 4.1 二级界面设计层

在现有 React、Ant Design、Stylus 和 Manate 状态管理之上增加统一的表现层。该层只负责：

- 设计变量与主题配色。
- 页面、卡片、表单、弹窗、菜单和状态组件的共同样式。
- 响应式布局与文字防挤压规则。
- 设置搜索索引和定位。
- 主题临时预览作用域。
- 可见文案的国际化解析与回退。

现有 store 方法、提交函数、测试连接函数、菜单动作回调、持久化逻辑和 IPC 调用保持原入口。

### 4.2 设计变量

二级界面使用语义变量，不在组件内散落固定色值。变量至少覆盖：

- 页面背景、普通表面、浮层表面、弱化表面。
- 主文字、次要文字、禁用文字。
- 默认边界、强调边界、焦点环。
- 主色及其弱化背景。
- 成功、信息、警告、错误和危险操作色。
- 小、中、大圆角。
- 卡片、菜单和弹窗阴影。
- 紧凑、默认两档间距与控件高度。

新的语义变量通过兼容适配器从已有 `uiThemeConfig` 推导。旧主题缺少新变量时，系统使用安全的派生值，不要求迁移现有用户数据。

### 4.3 终端隔离

终端背景使用独立常量 `#0E0F12`，不从 `uiThemeConfig.main` 或导入主题的 `terminal:background` 取值。应用主题、预览主题和导入主题在进入终端渲染前均归一化背景值。

终端颜色编辑器可以继续编辑前景、光标、选区和 ANSI 颜色。背景字段显示为已锁定状态，并解释“ShellPilot 在所有 UI 主题下保持近黑终端背景”。

## 5. 设置中心

### 5.1 信息结构

保留现有“顶部主分类 + 左侧子分类 + 右侧内容”的结构，降低学习成本：

- 顶部主分类保持连接、通用设置、UI 主题、快捷命令、配置模板和工具中心。
- 左侧显示当前主分类下的子分类，使用统一图标、文字和选中状态。
- 右侧内容使用页面标题、说明和语义卡片承载配置。
- 页面顶部显示设置搜索、语言入口、自动保存状态和关闭操作。

### 5.2 配置卡片

设置项按语义分组，例如“启动与连接”“界面与语言”“网络与更新”。每个字段包含：

- 清晰的字段名称。
- 必要时显示一行简短说明。
- 单位、默认值或推荐值提示。
- 就地帮助入口，而非只依赖长气泡。
- 与当前保存语义一致的反馈。

原本自动保存的设置继续自动保存；原本需要显式保存的表单继续显示保存按钮，不在视觉改造中统一改变保存语义。

### 5.3 设置搜索

设置搜索基于静态元数据索引页面标题、字段名称、别名和帮助关键词。搜索不索引字段当前值，不读取密码、Token、主机地址等敏感内容。

点击搜索结果只完成导航、展开和滚动定位，不修改配置。快捷入口使用 `Ctrl+K`，并提供鼠标入口。

## 6. UI 主题中心

### 6.1 主题库

主题中心以卡片形式展示配色、明暗类型、名称和简短说明，提供筛选、搜索、预览和应用。首批内置主题为：

- 海湾蓝：默认浅色专业主题。
- 翡翠绿：舒缓稳重的浅色主题。
- 云境紫：强调 AI 产品感的浅色主题。
- 暖砂橙：低压、温和的浅色主题。
- 石墨夜：保持清晰分层的深色主题。

主题卡片只展示外围 UI 配色。所有卡片中的终端预览都使用固定近黑背景。

### 6.2 临时预览与应用

点击“预览”后，主题仅作用于设置中心的预览作用域，不写入配置。用户点击“应用”后才调用现有主题保存入口。取消预览、关闭设置或发生渲染错误时恢复当前已保存主题。

预览区域至少包含配置卡片、表单控件、右键菜单、状态提示和终端样例，以便用户判断主题的整体可读性。

### 6.3 高级编辑

高级颜色编辑默认折叠。展开后显示本地化名称、英文名称和原始变量名，例如“主色 / Primary (`primary`)”。

内置主题为只读；用户可复制后编辑。导入、导出、自定义主题和原有文本编辑能力继续保留。第三方主题名称不翻译，避免破坏主题身份和导入导出一致性。

## 7. 配置表单与弹窗

### 7.1 共用容器

连接、模型 API 和同步配置使用统一容器规范：

- 顶部显示标题、上下文说明和关闭按钮。
- 复杂表单使用左侧步骤或分类导航。
- 内容按语义卡片分组。
- 字段错误就地显示在对应控件下方。
- 底部操作区在滚动时保持可见。
- 取消、保存、保存并执行等动作维持现有业务语义。

### 7.2 连接配置

连接配置按基础信息、身份验证、代理与跳板、SSH 隧道、启动脚本和高级选项分组。协议类型、连接名称和当前主机在标题区明确展示。

“测试连接”“取消”“保存”“保存并连接”等按钮继续调用现有动作。视觉层不得合并或替换原动作。

### 7.3 模型 API

模型 API 配置按服务商、地址与认证、模型、角色与语言、Agent Skill、MCP Server 和代理分组。长 URL、模型名和启动参数允许换行或横向输入框内部滚动，不撑破页面。

测试结果在操作区域就地显示，原有通知和错误细节继续保留。

### 7.4 同步与备份

先选择 WebDAV、GitHub Gist、Gitee Gist 或自定义服务器，再显示对应字段。数据选择、导入、上传、下载、测试连接和保存动作保持不变。

Token、密码和同步密钥继续使用现有安全输入和存储方式，视觉层不读取或复制敏感数据。

## 8. 应用内右键菜单

### 8.1 共用渲染规范

书签、终端、SFTP、标签页、传输列表和输入框菜单统一使用：

- 220–360px 自适应宽度。
- 统一图标列、文字列和快捷键列。
- 主要动作、编辑动作、辅助动作和危险动作分组。
- 删除、覆盖、清空等危险动作置底并标红。
- 禁用状态保留原因提示所需的可读对比度。
- 子菜单箭头、分隔线、悬停、焦点和键盘选中状态一致。

菜单文字可在两行内按完整单词或中文语义自然换行。快捷键列不压缩。菜单靠近视口边缘时自动翻转或偏移；高度超过视口时在菜单内部滚动。

### 8.2 动作保持

菜单项继续由现有构建函数生成，统一渲染器只消费菜单描述并调用原回调。项目不得因视觉分组而删除、重命名内部 action key 或改变触发条件。

## 9. 国际化

### 9.1 支持策略

客户端已有语言继续保留。ShellPilot 新增或定制的可见文案必须提供简体中文和 English。现有硬编码中文、英文、校验错误和按钮文字逐步替换为翻译键。

内置主题名称和说明提供双语；第三方与导入主题名称保持原文。SSH、SFTP、API、MCP、Agent、Token 等行业术语按产品词汇表统一，不进行生硬直译。

### 9.2 语言预览

语言选择支持设置中心内的临时预览。预览不保存；用户确认后沿用现有语言保存和界面刷新机制。取消预览恢复原语言。

### 9.3 回退顺序

翻译解析按以下顺序回退：

1. 当前语言的 ShellPilot 覆盖文案。
2. 当前语言的上游 Electerm locale 文案。
3. ShellPilot English 文案。
4. 可读的默认文案。

界面不得直接显示内部键名、`undefined`、空白按钮或乱码。所有本地文案文件和源码使用 UTF-8。

## 10. 响应式与文字防挤压

应用最小窗口为 `590×400`。二级界面遵循以下规则：

- 重要标题、字段名称和按钮不使用省略号；空间不足时换行或改为单列。
- 英文按完整单词换行，中文按自然语义换行。
- 控件不得覆盖文字，按钮不得压缩到不可读。
- 低于 820px 时，左侧子分类改为顶部选择器，表单统一单列。
- 低于 680px 时，设置搜索折叠为图标，顶部主分类横向滚动。
- 多个按钮优先换行；次要动作可以进入“更多”菜单，但主要提交动作始终可见。
- 页面本身不产生横向滚动；代码、URL 和长路径只在所属控件内部滚动或换行。

布局验收覆盖 `590×400`、`820×600`、`1100×700` 和 `1600×900`，并覆盖 100%、125% 和 150% 缩放。

## 11. 错误处理与回退

- 主题预览或应用失败时恢复当前已保存主题，并显示本地化错误。
- 导入主题存在非法颜色、缺失字段或不支持变量时，在保存前列出可理解的错误。
- 任何导入或自定义主题都在终端渲染前锁定背景为 `#0E0F12`。
- 翻译加载失败时使用既定回退顺序，不阻断页面渲染。
- 设置搜索索引失败时隐藏搜索增强，原有导航继续可用。
- 二级界面样式加载异常时，现有 Ant Design 控件与业务回调仍保持可操作。
- 原有连接测试、保存失败、同步失败和危险操作确认流程继续负责业务错误。

## 12. 验证策略

### 12.1 自动化验证

- 设计变量派生和旧主题兼容测试。
- 所有主题下的终端背景锁定测试。
- 主题预览、取消、应用和异常恢复测试。
- 语言解析、简中/英文覆盖和回退顺序测试。
- 可见硬编码文案扫描，阻止新增未国际化字符串。
- 设置搜索不索引敏感值的测试。
- 右键菜单项、action key、禁用条件和回调保持测试。
- 连接、模型 API、同步配置的现有单元测试和端到端测试回归。

### 12.2 布局与视觉验证

- 四档窗口尺寸、三档缩放和简中/英文组合检查。
- 长字段名、长错误信息、长 URL、长主题名和长菜单项压力测试。
- 键盘导航、焦点环、悬停、禁用和危险状态检查。
- 海湾蓝、翡翠绿、云境紫、暖砂橙和石墨夜的对比度检查。
- 终端标签区、画布和空白区域在全部主题中保持 `#0E0F12`。
- 主工作台布局和功能截图回归，确认本项目未改变主界面结构。

### 12.3 完成标准

满足以下条件才可认为实施完成：

1. 范围内页面和菜单使用统一设计规范。
2. 现有业务动作和保存语义无回归。
3. 简体中文和 English 无可见未翻译、乱码或空白文案。
4. 最小窗口与 150% 缩放下无文字覆盖、按钮裁切或页面横向滚动。
5. 所有主题下终端背景均为 `#0E0F12`。
6. 现有单元测试、相关端到端测试和新增主题/国际化/响应式测试通过。
