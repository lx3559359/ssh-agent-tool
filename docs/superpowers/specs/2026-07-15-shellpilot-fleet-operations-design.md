# ShellPilot 多服务器状态与模型健康检测设计

## 1. 目标

下一轮开发只处理四个彼此有清晰边界的交付：

1. 建立可重复的稳定性基线，阻止启动白屏和空数据崩溃回归。
2. 在最左侧导航增加“状态总览”，批量查看已保存 SSH 服务器的只读状态。
3. 把服务查询改为自动发现、搜索、多选和结构化结果，不要求用户输入完整服务名。
4. 自动检测当前 AI API 与所选模型的真实连接状态。

这四项完成前，不继续扩充高风险批量修改、自动修复或无人值守定时巡检。

## 2. 已选界面方向

采用用户确认的第一张预览图：

- 最左侧导航首项为“状态总览”，位于“服务器”上方。
- 中间主工作区使用密集表格，展示服务器、连接、CPU、内存、磁盘、负载、异常服务、防火墙、运行时间和最后检测。
- 右侧继续保留可拉伸 AI 助手。
- 顶部只保留分组、状态、搜索、刷新和更新时间。
- 服务器多选后显示只读批量工具条。
- 日间与夜间主题使用现有 ShellPilot 主题变量，不建立独立颜色体系。

“服务器”继续负责连接资料管理；“状态总览”只负责跨服务器观察、筛选和进入诊断。

## 3. 状态总览数据范围

### 3.1 服务器来源

状态总览读取现有书签中的 SSH 连接，不复制密码、私钥或连接配置。列表只显示：

- 书签名称、分组和标签
- 主机、端口和用户名
- 最近采集状态和时间
- 已脱敏错误原因

密码、API Key 和私钥内容不得进入状态快照、工具日志、AI 上下文和导出报告。

### 3.2 采集内容

第一阶段只读采集：

- SSH 可达性和握手耗时
- CPU、内存、Swap、负载和运行时间
- 挂载点和磁盘使用率
- 默认路由、网卡、CIDR 和 DNS
- 防火墙提供者与启用状态
- systemd/OpenRC/SysV 服务摘要
- Docker、Docker Compose、Supervisor 和 PM2 的存在性与摘要
- 异常服务数量及准确服务名

不采集完整日志正文。只有用户选择“查看日志”或“交给 AI 分析”时，才按范围读取最近日志。

### 3.3 性能约束

- 默认并发 5 台，可在后续设置中调整为 1 至 10 台。
- 单台连接超时 12 秒，单项探针超时 8 秒，总采集超时 30 秒。
- 状态缓存 60 秒；缓存有效时不重复连接。
- 用户可取消整体刷新，也可单独重试失败服务器。
- 离开状态总览后停止自动刷新和未开始任务。
- 默认仅手动刷新，不在应用启动时连接全部服务器。

## 4. 服务自动发现

服务状态不再使用“输入服务名并编辑命令”的默认流程。

打开服务选择器时：

1. 优先使用最近一次服务器状态快照。
2. 缓存不存在或过期时执行一次只读服务发现。
3. 按“系统服务、容器服务、进程管理器”分组。
4. 每项显示友好描述、准确名称、运行状态、子状态和是否自启动。
5. 支持搜索、状态筛选、单选、多选和“一键选择全部异常”。

选择后可执行：

- 查看状态
- 查看最近日志
- 查看端口和进程
- 交给 AI 分析

这些操作均为只读，不进行二次确认。原始命令放入折叠的“高级命令预览”，不作为普通入口。

## 5. AI API 与模型状态

状态必须区分“已经配置”和“真实可用”。统一状态为：

- `unconfigured`：缺少 API 地址或密钥
- `checking`：正在检测
- `reachable`：接口与鉴权可用，但模型尚未完成真实请求
- `available`：当前模型完成最小真实请求或最近一次对话成功
- `auth-error`：鉴权失败
- `model-error`：模型不存在或无权限
- `quota-error`：限流、余额不足或配额不足
- `network-error`：超时、DNS、TLS 或代理失败
- `stale`：检测结果超过有效期

自动检测触发条件：

- 打开 AI 助手且状态超过 5 分钟
- 切换 API 配置
- 切换模型
- 发送消息前状态已过期
- 实际对话成功或失败

只自动检测当前选中的配置和模型，不遍历全部模型。模型列表中显示各模型缓存状态；未检测模型显示灰色。用户可在 API 设置中手动执行“检测全部模型”，并发上限为 2。

## 6. AI 联动

状态总览向 AI 提供结构化、脱敏后的选中服务器快照。AI 入口包括：

- 分析异常服务器
- 对比选中服务器
- 为选中异常服务生成只读排查计划

AI 不会因为状态刷新自动发起对话。只有用户点击 AI 操作或发送问题后才产生模型请求。

## 7. 错误处理

- 单台失败不能中止其他服务器采集。
- 错误显示为可理解分类：连接超时、鉴权失败、主机密钥错误、权限不足、命令缺失、系统不支持。
- 探针返回空值时显示“未检测到”，不得把空值解释为正常。
- 所有组件必须接受空数组、空书签、已删除书签和取消中的任务。
- 渲染错误不得访问 `null.data`；空响应必须先规范化再进入 UI。

## 8. 验收标准

1. 启动、无书签、只有本地终端和 AI 未配置四种场景均不白屏。
2. 左侧“状态总览”可进入和退出，原 SSH/SFTP 标签不丢失、不重连。
3. 12 台模拟服务器可按并发 5 采集，单台失败不阻塞其余任务。
4. 表格支持筛选、多选、取消刷新和单台重试，窄窗口无文字竖排或遮挡。
5. 服务选择器能自动列出准确服务名并支持多选，纯查询不弹风险确认。
6. AI 顶部状态能自动从“检测中”变为准确结果，并显示检测时间和延迟。
7. 日间、夜间和 125% Windows 缩放下均可使用。
8. 单元测试、UI 回归、模拟多服务器测试和真实服务器只读回归全部通过后才进入打包。

## 9. 发布边界

计划阶段不修改版本号，也不发布在线更新。功能通过本地安装包和真实服务器只读回归后，再单独决定版本号并执行 GitHub、国内源和客户端更新验证。
