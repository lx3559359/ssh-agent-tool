# ShellPilot v0.4.0

## [新增]

- 新增统一 AI 运维安全事务：SSH 终端、AI/Agent、快捷命令和 SFTP 的已识别修改共用准备、确认、执行、验证、保留与回滚状态机。
- 新增服务器异常一键 AI 诊断：从异常服务、容器、平台或告警生成只读排查计划，确认后逐步执行，并显示实时进度、限长输出和取消入口。
- 新增安全操作中心四类记录，集中查看执行中、可回滚、历史记录和旧版记录，并提供取消任务、立即回滚、重试和保留修改操作。
- 新增 endpoint 与恢复绑定校验；断线重连后仅允许相同主机、端口和账号使用通过完整性验证的恢复资料。
- 新增 SFTP 删除、重命名、权限、编辑保存和 SFTP 传输保护，覆盖上传、覆盖、同端复制/移动及受限的跨端目标恢复。
- 新增 `docs/AI-OPS-SAFETY-TRANSACTIONS.md` 和本地安全 smoke，说明支持矩阵、不可恢复边界、性能策略、故障恢复与候选测试方法。

## [修复]

- 修复取消只读诊断后仍可能继续执行后续步骤的问题，取消会传递到当前远程命令并停止剩余计划。
- 修复安全记录在 endpoint 变化、恢复资料不完整或恢复绑定校验失败时仍可能展示可回滚操作的问题。
- 修复 SFTP 覆盖或移动只按文件大小判断结果的问题，改为校验受限内容身份和元数据，避免同大小内容被误判。
- 修复 SFTP 传输取消、重试或窗口关闭时任务状态可能无法安全收口的问题，并保留可恢复目标的审计入口。

## [改动]

- 服务器状态巡检和 AI 异常诊断保持按需执行、无后台轮询；探针限制并发、超时和输出大小，普通 SSH 终端只读输入不增加确认步骤。
- 已识别且存在恢复提供方的修改会先建立并验证恢复点；未知、动态或无法证明影响范围的命令只显示风险，不声称能够自动回滚。
- FTP 无自动回滚。FTP 覆盖、删除和修改仍需用户自行备份，不能使用 SFTP 安全事务的恢复承诺。
- `vim/nano` 内部写入不能可靠快照；交互编辑器、脚本解释器、alias、function、复杂多行命令和第三方程序内部副作用不提供自动恢复保证。
- 在线更新批准机制保持不变：本候选不会修改批准清单，不会仅因版本号或发布资产存在就向客户端推送更新。
