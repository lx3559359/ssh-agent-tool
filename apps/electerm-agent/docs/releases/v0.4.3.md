# ShellPilot v0.4.3

## [新增]

- 新增多服务器状态工作台，可批量查看已接入服务器的在线状态、资源使用、网络、防火墙、服务、容器和平台进程。
- 新增服务器服务自动识别与按需检查，可从当前服务器发现 systemd 服务、容器及常见平台组件，再选择需要关注的项目。
- 新增模型 API 自动健康检测，区分可用、待检测、认证失败、额度不足、模型不可用和网络异常等状态。

## [修复]

- 修复 AI 助手每次提问都被当作首次对话的问题；连续提问现在会携带同一终端会话的历史上下文。
- 修复用户提问被折叠后难以查找的问题；用户消息和助手回答按时间顺序完整显示。
- 修复关闭并重新打开 AI 助手后历史记录顺序或内容丢失的问题。
- 修复流式响应中的中文字符分片、停止生成、重试和异步响应竞态问题。
- 修复部分中转站返回额度不足的 HTTP 403 时被错误显示为“认证失败”的问题。
- 修复流式回答尚未完成时模型状态被过早标记为“可用”的问题，失败结果会按额度、认证、模型和网络异常分类。
- 修复客户端异常退出后，无会话标识的 AI 请求可能永久停留在“运行中”的问题。
- 修复本地 CLI 工具参数可能在结果和对话历史中暴露敏感值的问题。
- 修复 AI 引用 SFTP 文件、Agent 工具和 MCP/CLI 响应时可能污染或覆盖对话上下文的问题。

## [改动]

- AI 上下文按终端会话隔离，并设置单会话和全局历史上限，避免长时间使用造成无界内存增长。
- 发送给模型的历史消息统一经过长度限制、敏感凭据清理和异常数据过滤。
- 完善 AI 请求、Agent、本地 CLI 和 SFTP 上下文的取消与清理流程，关闭面板不再误删已经完成的聊天记录。
- 补齐发布后完整自检所需的前端解析器和串口模拟测试依赖。
