# ShellPilot v0.4.4

## [新增]

- 新增统一操作链路编号，可关联 SSH、SFTP、AI、在线更新、安全操作和回滚记录，便于从工具日志定位一次完整操作。
- 新增启动时间、首个终端可用时间、AI 首字延迟和内存占用等性能指标，采用有界采样，不进入终端与文件传输热路径。
- 新增异常退出恢复提示，可恢复未连接的标签外壳并汇总中断任务；恢复后不会自动连接服务器、执行命令或继续修改操作。
- 新增本地 SSH/SFTP/AI/更新/回滚端到端测试、崩溃恢复测试、性能基线测试和环境变量驱动的真实服务器回归测试。

## [修复]

- 修复 pnpm 依赖布局下 `cpu-features` 缺少构建配置导致 Windows 安装包无法重建原生模块的问题，并补充打包后运行时与 SQLite 初始化冒烟验证。
- 修复设置中心同时加载全部页签导致的启动等待与无关模块加载问题，现在只加载当前页签。
- 修复异常退出后恢复流程可能被 Electron 子进程占用配置文件阻塞的问题，Windows 测试与恢复清理会正确结束进程树。
- 修复单实例启动测试、设置语言预览测试和响应式布局测试与当前组件加载契约不一致的问题。
- 修复质量测试使用不同模块格式产生的 Node.js 启动警告。

## [改动]

- 恢复快照仅保存非敏感标签状态和任务摘要，不保存密码、私钥、API Key、终端正文、命令或附件内容。
- 性能指标和链路日志采用限量、脱敏和失败降级策略，普通 SSH 输入、终端渲染与 SFTP 数据传输保持原有轻量路径。
- 新增 `test-quality-e2e`、`test-performance-e2e` 和 `test-real-server-e2e` 发布前验证命令。
- 真实服务器回归只执行只读 SSH 命令，SFTP 写入限定在随机隔离目录并在测试结束后清理。
