# ShellPilot v0.3.9

## [新增]

- 正式客户端顶部新增“服务器状态”入口，仅在当前 SSH 会话已连接时启用。
- 新增只读服务器状态中心，集中查看系统与资源、systemd 服务、网卡与 IP、路由与 DNS、监听端口、防火墙与安全模块、Docker/Podman 容器等信息。
- 新增服务器平台与软件服务自动识别，支持按服务名、安装路径和 Docker Compose 项目归组，并可添加本地自定义识别规则。
- 新增状态结果复制、Markdown/JSON 导出和显式发送给 AI 助手分析；导出与 AI 上下文均进行凭据脱敏和内容限长。

## [修复]

- 修复服务器状态入口未随当前真实 SSH 会话连接状态刷新的问题。
- 修复 CentOS 7 等环境下 systemd 服务列表读取失败的问题。
- 修复防火墙返回 `not running` 时被误判为已启用的问题。
- 修复检测失败或超时时整体状态仍可能显示为正常的问题。
- 修复服务启动参数和原始检测输出可能在导出时携带密码、Token 或 API Key 的问题。
- 修复探针返回的 Docker Compose 项目无法参与平台识别的问题。
- 修复状态探针超时后只停止前端等待、远程 SSH 命令仍可能继续运行的问题。

## [改动]

- 所有状态探针固定为只读命令，限制并发数、执行时长和返回内容大小；不会自动轮询，也不会执行修改操作。
- 帮助中心和离线中文使用手册补充服务器状态中心的入口、检测范围、平台识别、导出与 AI 分析说明。
- 真实服务器回归流程新增 7 项只读状态扫描和前后指纹校验，确保检测过程不改变服务器配置。
