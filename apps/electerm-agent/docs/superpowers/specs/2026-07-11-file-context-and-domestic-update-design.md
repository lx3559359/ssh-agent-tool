# ShellPilot 文件上下文与国内更新源设计

日期：2026-07-11

## 目标

本设计解决三个已确认问题：

1. 长日志和压缩日志无法完整、连续、安全地读取。
2. AI 助手缺少本地上传、剪贴板粘贴和 SFTP 拖放附件能力。
3. GitHub 在国内检查和下载安装包较慢，缺少国内更新镜像。

本期不改变 SSH 终端和 SFTP 传输的既有核心行为，不发布未经用户确认的更新。

## 总体方案

新增统一的“文件资源管道”。本地文件、剪贴板文件和 SFTP 文件先转换为统一资源描述，再由预览器、日志读取器、压缩包读取器和 AI 上下文构建器按需消费。

更新模块采用“魔搭优先、GitHub 备用”的双源策略。源代码仍以 GitHub 为主，魔搭仅镜像用户确认发布的正式更新资产。

## 文件资源模型

每个附件使用统一结构：

- `id`：会话内唯一标识。
- `name`：显示名称。
- `source`：`local`、`clipboard` 或 `sftp`。
- `path`：本地路径或远程路径，不直接写入模型消息。
- `size`：原始文件大小。
- `mimeType`：可识别时记录 MIME 类型。
- `kind`：`text`、`archive`、`image`、`document`、`binary` 或 `unknown`。
- `status`：`pending`、`indexing`、`ready`、`failed`。
- `reader`：只读能力标识，例如区间读取、尾部读取、搜索、压缩成员读取。
- `metadata`：编码、行数、压缩成员、截断信息和错误信息。

资源对象只保存元数据和读取句柄，不默认把完整文件放入 React 状态或 AI 历史记录。

## 长日志读取

### 普通文本日志

支持本地和 SFTP 文件的分块读取，不再把 64 KB 预览上限当作完整文件读取上限。

首期能力：

- 读取开头、结尾和指定字节区间。
- 按行显示并继续加载上一段或下一段。
- 按关键词搜索，返回命中行及其前后上下文。
- 显示文件大小、当前读取范围、是否还有更多内容。
- UTF-8 多字节字符边界安全处理。
- 文件持续增长时支持刷新尾部，不自动覆盖用户当前阅读位置。

默认单次读取 256 KB，允许内部按需连续读取，但 UI 和 AI 请求都必须分块，避免一次占用过多内存。

### 压缩日志

首期支持：

- `.gz`：流式解压单个内容。
- `.zip`：列出成员后读取选中的文本成员。
- `.tar.gz`：流式遍历并读取选中的文本成员。

压缩包默认只读，不在远程服务器执行解压命令，也不修改远程目录。需要临时文件时使用客户端专属临时目录，并在会话结束或超时后清理。

安全限制：

- 阻止绝对路径和 `..` 路径穿越。
- 限制成员数量、单成员解压大小和总解压大小。
- 拒绝符号链接、设备文件和可执行内容的自动解析。
- 二进制成员只显示信息，不直接发送给文本模型。
- 压缩率异常时中止读取并给出中文错误。

建议默认限制：最多 5000 个成员、单成员 100 MB、单次分析解压总量 500 MB。设置页可降低限制，不提供无限制选项。

## AI 附件交互

AI 输入区增加附件按钮和附件队列，支持：

- 文件选择器选择一个或多个本地文件。
- 在输入框粘贴剪贴板中的文件、图片或文本。
- 从 Windows 资源管理器拖入文件。
- 从 ShellPilot SFTP 文件列表拖入远程文件。
- 继续保留“引用当前 SFTP 文件”快捷操作。

附件以紧凑条目显示名称、大小、来源、状态和移除按钮。拖入目录默认拒绝，后续可以单独设计目录批量分析。

发送消息时只提交用户问题和附件引用。模型需要内容时，由上下文构建器按类型读取：

- 小文本可直接附加。
- 大日志先生成索引，再按关键词或区间读取相关片段。
- 压缩包先列出成员，再选择相关文本成员。
- 图片交给支持视觉输入的模型；不支持视觉时明确提示。
- Word、PDF 等文档通过后续可插拔解析器接入，首期先保存附件并提示当前解析能力。

附件不会自动上传到任何第三方存储。只有用户发送消息时，必要内容才会发送给当前配置的模型 API。

## AI 大文件分析流程

1. 收集文件元数据和用户问题。
2. 判断文件类型、编码和可用读取能力。
3. 对长日志建立轻量索引，包括大小、时间范围、关键词命中和错误级别统计。
4. 根据问题选择相关片段，单轮控制在模型上下文预算内。
5. 模型可请求继续读取指定区间或压缩成员。
6. 最终回答标注使用了哪些文件和范围，并提示未读取部分。

该流程禁止将“提高字符上限”作为完整日志支持方案。

## 国内更新源

### 仓库

已创建公开魔搭仓库：

`https://modelscope.cn/models/lx3559359/ShellPilot-Updates`

仓库仅存放正式更新资产，不作为源代码主仓库。

### 更新资产

每个正式版本至少包含：

- `latest.yml`
- `shellpilot-update.json`
- `aigshell-update.json`（兼容现有旧客户端）
- Windows 安装包
- 安装包对应的 `.blockmap`
- `checksums.json`

两个审批清单内容保持一致，包含产品名、版本、通道、发布时间、发布批准状态、兼容产品和各文件 SHA256。新客户端优先读取 `shellpilot-update.json`，找不到时回退到 `aigshell-update.json`。

### 检查与下载顺序

1. 客户端先请求魔搭上的轻量更新索引。
2. 验证索引结构、产品名、更新通道、版本号和发布批准状态。
3. 从魔搭下载更新资产；支持超时、断点续传和进度显示。
4. 魔搭检查失败、下载失败或校验失败时自动切换 GitHub Releases。
5. 两个源都失败时保留当前版本并显示可操作的中文错误，不启动安装。

客户端不需要安装 ModelScope SDK，也不需要用户登录魔搭。公开仓库文件通过 HTTPS 下载。

### 发布控制

发布分为“构建”和“批准同步”两个阶段：

1. 构建阶段生成并测试资产，不修改线上更新索引。
2. 用户明确确认发布后，上传 GitHub 和魔搭的版本文件。
3. 两边文件完整且校验一致后，最后上传批准清单和最新版本索引。
4. 任一平台同步失败时，不把该平台标记为可更新。

这样可避免开发过程中的每次构建被客户端识别为正式更新。

## 模块边界

- `resource-descriptor`：统一附件模型和来源适配。
- `range-reader`：本地与 SFTP 区间读取接口。
- `archive-reader`：压缩成员枚举与安全读取。
- `log-indexer`：长日志搜索、时间和级别索引。
- `ai-attachment-store`：附件队列和会话生命周期。
- `ai-context-provider`：根据模型预算按需组装内容。
- `update-source`：统一魔搭与 GitHub 更新源接口。
- `release-sync`：正式发布资产的双平台同步和校验。

各模块通过结构化数据交互，UI 不直接执行解压、网络下载或大文件读取。

## 错误处理和日志

所有失败必须记录到 ShellPilot 工具日志，敏感路径、账号、API Key 和密码继续脱敏。

用户可见错误需要包含：

- 失败对象，例如文件、压缩成员或更新源。
- 失败阶段，例如读取、解压、解析、下载或校验。
- 可执行建议，例如重试、切换源、缩小范围或检查权限。

单个附件失败不应清空输入框或移除其他附件。单个更新源失败不应阻止备用源继续检查。

## 测试与验收

### 文件与日志

- 读取超过 1 GB 的稀疏测试日志时，内存不随文件大小线性增长。
- 可连续读取开头、中间和结尾，UTF-8 中文不乱码、不切半字符。
- `.gz`、`.zip`、`.tar.gz` 能列出并读取文本内容。
- 路径穿越、压缩炸弹、二进制文件和损坏压缩包被安全拒绝。

### AI 附件

- 本地选择、粘贴、资源管理器拖放和 SFTP 拖放均生成正确附件。
- 删除、失败重试和发送后状态正确。
- 大日志只发送相关片段，并显示使用范围。
- 未发送消息前不向模型 API 传输附件内容。

### 更新

- 魔搭正常时使用魔搭检查和下载。
- 魔搭不可用时自动回退 GitHub。
- 未批准版本、哈希不一致、缺少 `.blockmap` 或产品名错误时拒绝更新。
- 发布脚本在用户未确认时不会改动线上最新版本索引。

## 实施顺序

1. 长日志区间读取和压缩包安全读取。
2. AI 附件队列、本地选择与粘贴。
3. SFTP 拖放到 AI 输入区。
4. AI 按需分块分析和范围引用。
5. 魔搭/GitHub 双更新源。
6. 双平台发布同步、回退和端到端验收。
