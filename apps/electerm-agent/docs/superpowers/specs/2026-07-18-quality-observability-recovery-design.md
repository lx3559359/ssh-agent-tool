# ShellPilot 质量、可观测性与崩溃恢复设计

## 背景

ShellPilot 已具备 SSH、SFTP、AI 助手、在线更新、安全操作与回滚能力，但这些能力目前主要依靠各模块自己的标识和日志。发生跨模块故障时，仍需要人工拼接前端日志、SSH 会话、Agent 任务、更新记录和安全操作记录。同时，现有自动测试缺少覆盖真实 SSH/SFTP、更新、AI 流式响应和回滚全过程的统一端到端测试，客户端也缺少稳定、可比较的性能基线与异常退出恢复机制。

本轮采用“统一观测内核 + 分层 E2E + 保守恢复”方案，在不改写终端核心、不自动重放高风险操作的前提下补齐质量基础设施。

## 目标

1. 建立 SSH、SFTP、AI、更新和回滚的端到端自动测试，并区分本地自动回归与发布前真实服务器回归。
2. 为一次用户操作生成统一链路编号，使前端、主进程、SSH/SFTP、Agent、更新和安全事务日志可以关联查询。
3. 建立启动时间、首个终端可用时间、AI 首字延迟和内存占用指标，并形成可比较的本地基线。
4. 异常退出后恢复标签页、布局和任务摘要，同时保证不自动重连、不自动执行命令、不自动安装更新、不自动回滚。

## 非目标

- 不替换 Electerm 原有 SSH/SFTP 会话实现。
- 不在终端输入、输出的高频数据通道中逐字符写结构化日志。
- 不采集或上传云端遥测。
- 不保存密码、私钥、API Key、终端屏幕正文或附件正文到恢复快照。
- 不在启动恢复阶段自动继续任何远程操作。
- 不把真实测试服务器凭据写入仓库、构建产物、测试截图或日志。

## 总体架构

新增四个相互独立但共享事件模型的基础模块：

1. `operation-context`：生成、传播和序列化统一链路上下文。
2. `quality-log`：在现有 `electron-log` 和脱敏层之上写入结构化链路字段。
3. `performance-metrics`：记录有限的性能事件和低频内存快照。
4. `recovery-snapshot`：管理运行状态快照、正常退出标记和异常恢复提示。

这些模块只通过明确的生命周期事件接入现有代码，不改变 SSH 数据协议、SFTP 传输协议、AI 请求格式和更新资产格式。

## 统一链路编号

### 数据模型

每次用户主动操作创建一个 `traceId`，格式为 `sp-<时间戳>-<随机短码>`。其余现有编号保持不变，并作为链路的子标识：

```js
{
  traceId,
  operationId,
  taskId,
  requestId,
  sessionId,
  tabId,
  module,
  action
}
```

- `traceId`：贯穿一次用户意图，例如“检查更新”“发送 AI 问题”“上传文件并分析”。
- `operationId`：安全事务或回滚事务编号，继续使用现有事务存储生成的编号。
- `taskId`：Agent、诊断或后台任务编号。
- `requestId`：单次 IPC、AI 或网络请求编号。
- `sessionId`：SSH、AI 流或更新会话编号。
- `tabId`：当前客户端标签页编号，仅用于本地关联。
- `module`：`frontend`、`ssh`、`sftp`、`agent`、`update`、`rollback` 等固定模块名。
- `action`：稳定、可搜索的动作名，不使用用户输入作为动作名。

### 传播规则

- 渲染进程发起动作时创建 `traceId`，通过 IPC 参数传递给主进程。
- 主进程创建子请求时沿用 `traceId`，仅生成新的 `requestId`。
- 安全事务创建后，把 `operationId` 回填到同一链路上下文。
- 后台任务、Agent 任务和 AI 流分别回填 `taskId`、`sessionId`。
- 兼容未携带上下文的旧调用：日志层为该事件创建独立 `traceId`，不阻断原操作。

### 日志格式与安全

结构化日志至少包含：时间、级别、模块、动作、阶段、结果、耗时和链路字段。日志正文继续经过现有脱敏层，额外禁止记录以下内容：

- 密码、私钥、API Key、认证 Header 和 Cookie。
- 终端完整输入、终端完整输出和远程文件正文。
- 本地用户目录、临时目录完整路径；诊断导出仅保留脱敏路径。
- 用户聊天正文和附件正文；只允许记录长度、类型和处理结果。

日志查看器增加按 `traceId` 搜索能力，复制诊断摘要时输出链路编号、版本、模块、阶段和脱敏错误，不输出原始堆栈。

## 性能指标

### 指标定义

- `app_start_ms`：主进程启动到首个窗口完成加载。
- `first_window_interactive_ms`：主进程启动到主界面配置加载完成且可响应操作。
- `first_terminal_ready_ms`：主进程启动到首个终端可接收键盘输入；本地终端和远程 SSH 分开标记类型。
- `ai_first_token_ms`：AI 请求发出到首个有效文本片段到达。
- `ai_total_ms`：AI 请求发出到流完成、取消或失败。
- `memory_main_mb`：主进程常驻内存。
- `memory_renderer_mb`：主渲染进程常驻内存。
- `memory_total_mb`：当前应用进程树总内存。

### 采集原则

- 启动与 AI 指标采用事件打点，不使用轮询。
- 内存默认在启动稳定后采集一次，之后最多每 60 秒采集一次；窗口隐藏或应用空闲时不提高采样频率。
- 单次运行只保留必要聚合值和最近事件，最多保留 30 天或 1000 条，先到者触发清理。
- 指标保存在本机用户数据目录，不上传网络，不包含服务器地址、账号、命令和聊天内容。
- 测试报告以当前主机最近稳定基线为参照，先告警后阻断；连续建立基线后再启用发布阈值。

### 性能回归规则

第一阶段只记录基线和趋势。数据稳定后使用相对阈值：

- 启动和首终端可用时间相对基线恶化超过 20% 时告警。
- AI 首字延迟使用本地模拟服务测试，恶化超过 25% 时告警。
- 空闲内存相对基线增长超过 20% 且超过固定缓冲值时告警。

真实模型和真实服务器的网络延迟不作为阻断发布的唯一依据。

## 崩溃恢复

### 恢复快照

快照采用版本化结构，写入时使用临时文件加原子替换：

```js
{
  schemaVersion,
  appVersion,
  savedAt,
  cleanExit,
  activeTabId,
  layout,
  tabs: [],
  pendingTasks: []
}
```

每个标签只保存恢复所需的最小信息：标签类型、书签 ID、显示标题、布局位置、是否连接过和非敏感视图状态。临时快速连接如果没有书签，只保存可用于重新填写表单的非敏感字段，不保存密码、私钥内容或口令。

### 写入时机

- 标签新增、关闭、切换或布局变化后防抖写入。
- Agent、安全事务、更新任务状态变化后写入任务摘要。
- 正常退出前写入 `cleanExit: true`。
- 启动时先把当前运行标记为未正常结束；只有正常退出流程完成后才改回正常。

### 恢复行为

- 正常退出：沿用用户现有启动会话设置，不显示崩溃提示。
- 异常退出：恢复标签和布局，但 SSH/SFTP 标签处于“待重连”状态。
- 本地终端：恢复标签类型，不恢复上次终端进程和屏幕缓冲。
- 未完成任务：显示一次汇总提示，并提供“查看安全中心”“查看更新中心”“忽略”入口。
- 安全事务：沿用现有孤儿任务恢复逻辑，标记为中断或结果未知。
- AI 流：标记为中断，不继续请求；保留已持久化的消息和可重新发送入口。
- 更新：已下载但未安装的更新仅提示，不自动安装；正在下载的任务标记中断，可由用户重新检查。

### 绝对禁止自动恢复的动作

- 自动发送 SSH 命令。
- 自动恢复 SFTP 上传、覆盖、删除或重命名。
- 自动重启服务、修改防火墙、修改网络或权限。
- 自动执行回滚脚本。
- 自动继续 AI Agent 工具调用。
- 自动安装或重启应用完成更新。

## 端到端测试

### 第一层：本地自动回归

本地测试在隔离的用户数据目录运行，使用可控测试服务和临时文件：

| 模块 | 核心场景 |
| --- | --- |
| SSH | 密码连接、命令执行、`Ctrl+C`、断线、重连、关闭标签 |
| SFTP | 列表、上传、下载、重命名、覆盖确认、备份、失败清理 |
| AI | 配置、流式首字、上下文连续、取消、超时、有限重试、附件限制 |
| 更新 | 检查、无更新、有更新、源回退、下载进度、取消、审批门槛 |
| 回滚 | 创建临时文件、修改、生成事务、快捷回滚、结果与审计记录 |
| 恢复 | 强制结束客户端、重启、恢复标签、提示未完成任务、确认没有自动执行 |

本地 SSH/SFTP 服务使用测试专用账号和临时根目录；模拟 AI 与更新服务只监听本机。安装器替换测试只在隔离的打包任务中执行，普通 E2E 不覆盖当前开发安装。

### 第二层：发布前真实服务器回归

真实服务器参数只从环境变量读取：

- `SHELLPILOT_E2E_SSH_HOST`
- `SHELLPILOT_E2E_SSH_PORT`
- `SHELLPILOT_E2E_SSH_USER`
- `SHELLPILOT_E2E_SSH_PASSWORD` 或测试私钥路径
- `SHELLPILOT_E2E_REMOTE_ROOT`

真实回归只在专用远程目录内创建带随机编号的测试文件，结束时清理。默认只执行只读 SSH 命令；修改与回滚测试限制在该目录，不修改系统网络、防火墙、服务和用户配置。

真实回归覆盖：

1. SSH 建连、命令回显、交互中断和重连。
2. SFTP 上传、下载、校验哈希、重命名、备份和清理。
3. AI 引用终端输出并收到模拟或明确配置的模型响应。
4. 更新检查和更新源回退，不自动安装未批准版本。
5. 在测试目录中完成文件修改、事务记录和一键回滚。

### 测试隔离与产物

- 每次运行使用独立用户数据目录、独立端口和唯一 `traceId`。
- 失败时保存脱敏日志、截图、性能摘要和链路摘要。
- 测试失败不保留密码、API Key、远程文件内容或完整本地路径。
- 清理失败要在报告中单独列出，避免把环境污染误判为功能成功。

## 故障处理

- 链路日志失败不得阻断 SSH、SFTP、AI 或更新操作；降级到普通脱敏日志。
- 指标写入失败仅记录一次警告并停用本次运行的指标落盘，避免日志循环。
- 恢复快照损坏时移入隔离备份并启动默认界面，不出现白屏。
- 快照版本过旧时只恢复能安全识别的字段，无法识别的标签跳过并提示数量。
- E2E 测试服务启动失败时明确标记“测试环境失败”，不误报产品功能失败。

## 实施顺序

1. 为链路上下文、结构化日志、指标记录和恢复快照编写单元测试。
2. 实现统一链路上下文，并接入前端、SSH/SFTP、Agent、更新和回滚入口。
3. 实现性能事件采集与本地有界存储。
4. 实现恢复快照、正常退出标记、标签恢复和未完成任务提示。
5. 增加本地 SSH/SFTP、AI、更新、回滚和崩溃恢复 E2E。
6. 增加环境变量驱动的真实服务器回归套件和脱敏报告。
7. 运行完整单元测试、Lint、生产构建、打包冒烟和双层 E2E。

## 验收标准

- 任一 SSH、SFTP、AI、更新或回滚操作可以通过同一个 `traceId` 关联前端和主进程日志。
- 日志和测试产物不包含密码、API Key、私钥、聊天正文或完整本地路径。
- 可以查看四项核心性能指标及内存指标，并能与本机历史基线比较。
- 强制结束客户端后重新打开，标签与布局可恢复，未完成任务有明确提示。
- 恢复过程不自动连接服务器、不发送命令、不传输文件、不安装更新、不执行回滚。
- 本地自动回归覆盖五类主流程和崩溃恢复流程。
- 真实服务器回归可通过环境变量启用，凭据不会写入仓库或日志。
- 完整测试、Lint、生产构建和打包冒烟通过，SSH 终端基本输入、回车和 `Ctrl+C` 行为无回归。
