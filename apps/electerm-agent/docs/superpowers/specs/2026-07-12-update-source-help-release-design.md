# ShellPilot 更新源、帮助中心与发布说明设计

## 目标

让 Windows 客户端用户可以主动选择在线更新源，在客户端内查看完整中文帮助，并让每次发布都明确展示新增、修复和改动内容。

## 更新源

- 配置项 `updateSource` 支持 `auto`、`modelscope`、`github`。
- `auto` 按 ModelScope 国内源、GitHub 的顺序自动回退。
- 手动选择具体源时只访问该源；失败后提示用户切换更新源。
- 设置页和更新中心都提供选择控件，选择后立即持久化。
- 渲染进程版本检查和主进程原生增量更新使用同一选择规则。

## 帮助中心

- 顶部工具栏增加“帮助”按钮并打开客户端内模态窗口。
- 内容覆盖连接管理、SSH 终端、SFTP、安全备份恢复、AI/API、快捷命令、MCP/CLI、在线更新和日志排障。
- 仓库同时提供 `docs/USER_GUIDE_ZH.md`，便于离线查看和后续维护。

## 发布说明

- 每个版本在 `docs/releases/vX.Y.Z.md` 保存说明。
- 内容固定包含 `[新增]`、`[修复]`、`[改动]` 三个标题。
- GitHub 发布脚本优先读取该文件，客户端更新中心继续展示 Release 正文。

## 验证

- 单元测试验证更新源筛选、默认值、设置与更新中心控件。
- 单元测试验证帮助入口、帮助主题和发布说明格式。
- 完整单元测试、编译、Windows 打包、打包冒烟测试、发布资产验证和在线源验证必须通过。
