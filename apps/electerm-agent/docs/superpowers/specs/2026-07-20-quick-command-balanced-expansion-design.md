# ShellPilot 快捷命令均衡扩充设计

## 1. 背景

ShellPilot 已具备系统概览、磁盘、内存、端口、网络、服务、日志、Nginx、Docker、防火墙、抓包和服务器 IP 修改等快捷命令。当前命令主要集中在单个大型文件中，继续直接追加会提高维护成本，也容易让表单、风险控制和回滚逻辑产生分叉。

本轮采用“方案 B：按领域拆分命令目录 + 局部组件化”，在不修改终端核心、SSH 输入链路、SFTP 和 AI 对话逻辑的前提下，均衡扩充常见服务器维护与排查能力。

## 2. 目标

1. 新增 12 项只读排查能力和 6 项表单化维护能力。
2. 自动识别服务、网卡、容器、时区等对象，减少手工输入全称。
3. 所有修改类操作统一执行检测、校验、备份、确认、执行、验证和回滚流程。
4. 保留现有快捷命令 ID、收藏、历史记录和安全中心关联能力。
5. 支持常见 Linux 发行版，并在系统工具缺失时提供可理解的降级结果。
6. 改善快捷命令在窄窗口、日间主题和夜间主题下的可读性。

## 3. 非目标

1. 不替换 Electerm 终端、SSH 或 SFTP 底层实现。
2. 不允许快捷命令绕过安全事务直接执行修改。
3. 不在真实生产服务器上自动执行破坏性验证。
4. 不引入持续驻留的服务器监控进程。
5. 不将所有 Linux 管理操作一次性纳入本轮。

## 4. 功能范围

### 4.1 新增只读排查命令

| 领域 | 功能 | 主要结果 | 降级策略 |
| --- | --- | --- | --- |
| 系统 | CPU 负载与压力 | load、CPU 使用、运行队列、压力指标 | `mpstat` 不存在时使用 `top`、`vmstat`、`/proc/pressure` |
| 存储 | 磁盘 I/O 延迟与繁忙度 | await、util、队列和吞吐 | `iostat` 不存在时使用 `/proc/diskstats` 与 `vmstat` |
| 存储 | inode 与只读挂载 | inode 使用率、只读文件系统 | 使用 `df -i`、`findmnt`，缺少 `findmnt` 时使用 `/proc/mounts` |
| 存储 | 已删除但仍占用的文件 | 进程、文件描述符、占用空间 | `lsof` 不存在时扫描 `/proc/*/fd` |
| 系统 | 内核、OOM 与硬件异常 | OOM、I/O、文件系统和硬件错误 | 使用 `journalctl -k`，不可用时使用 `dmesg` |
| 系统 | 启动、重启与异常关机 | boot、reboot、shutdown 和崩溃记录 | 使用 `last`、`journalctl --list-boots` |
| 网络 | 网卡丢包、错误与流量 | RX/TX、drop、error、速率和链路 | `ethtool` 不存在时使用 `ip -s link`、`/sys/class/net` |
| 网络 | TCP 状态与异常连接 | TCP 状态分布、连接来源、重传线索 | `ss` 不存在时使用 `netstat` |
| 网络 | 路由、链路与 MTU | 默认路由、策略路由、网卡 MTU | 使用 `ip route`、`ip rule`、`ip link` |
| 安全 | SSH 认证失败与安全事件 | 登录失败、来源 IP、近期安全日志 | 自动选择 `journalctl`、`auth.log` 或 `secure` |
| 服务 | Cron 与 systemd Timer | 定时任务、下次执行、失败状态 | 非 systemd 系统仅展示 cron 目录与用户任务 |
| 容器 | Docker 健康、重启与存储 | health、restart、磁盘占用和异常容器 | Docker 不存在时返回“未安装或无权限”，不作为执行错误 |

### 4.2 新增表单化维护命令

| 功能 | 自动识别 | 表单参数 | 回滚方式 |
| --- | --- | --- | --- |
| 修改主机名 | 当前主机名、发行版 | 新主机名、是否同步 hosts | 恢复原主机名和 hosts 备份 |
| 管理 `/etc/hosts` | 当前条目 | IP、主机名、操作方式 | 恢复带时间戳的原文件 |
| 修改时区 | 当前时区、可用时区 | 目标时区 | 恢复原时区 |
| 管理 Swap | 当前 Swap、磁盘空间 | 创建、启用、禁用、大小、路径 | 恢复 fstab 并移除或重新启用对应 Swap |
| 服务开机策略 | 已发现服务与当前状态 | 服务多选、启用或禁用 | 恢复原 enable/disable 状态 |
| 管理定时任务 | 当前用户与系统任务 | 新增、禁用、删除、表达式、命令 | 恢复原 crontab 或 timer 文件 |

现有防火墙表单同时增强以下参数：开放或关闭、端口、协议、来源 CIDR、永久或临时规则。执行前记录实际防火墙实现和原规则，并生成对应回滚。

## 5. 目录与模块设计

现有 `server-maintenance-commands.js` 调整为兼容聚合入口，不再承载所有命令定义：

```text
src/client/components/quick-commands/
  server-maintenance-commands.js
  server-maintenance/
    index.js
    system.js
    storage.js
    network.js
    security.js
    services.js
    containers.js
    shared/
      discovery.js
      validation.js
      command-builders.js
      safety-metadata.js
```

职责说明：

- `index.js`：按固定顺序聚合各领域命令，校验 ID 唯一性。
- 领域模块：只声明命令元数据、探测逻辑、表单和命令构建器。
- `discovery.js`：封装发行版、init 系统、服务、网卡、容器、时区和工具可用性探测。
- `validation.js`：校验 IP、CIDR、端口、服务名、主机名、路径、时间表达式和枚举参数。
- `command-builders.js`：使用经过校验的参数生成命令，禁止直接拼接未经验证的用户输入。
- `safety-metadata.js`：统一生成风险等级、备份、验证、回滚和审计元数据。

现有命令 ID 不变。新增命令使用稳定、语义化 ID；测试阻止重复 ID 和无意变更。

## 6. 执行链路

### 6.1 只读命令

```text
读取 SSH 上下文
  -> 探测发行版和工具
  -> 选择主命令或降级命令
  -> 预览
  -> 执行
  -> 结构化显示结果和缺失工具提示
  -> 记录完成状态
```

只读命令可以直接执行，不创建回滚任务。探测失败时展示原因，不能把失败误标为仍在执行。

### 6.2 修改命令

```text
读取当前配置
  -> 自动填充表单
  -> 参数校验和磁盘预检
  -> 生成变更摘要
  -> 创建备份和回滚脚本
  -> 二次确认
  -> 执行修改
  -> 验证预期状态
  -> 写入安全中心
  -> 用户保留修改或快捷回滚
```

任何一个前置步骤失败都不得进入修改阶段。执行成功但验证失败时，安全中心突出显示“建议立即回滚”。连接中断时保留任务、备份位置和回滚入口。

## 7. 安全设计

1. 所有表单值按字段类型验证，并对 shell 参数进行安全引用。
2. 禁止在命令模板中使用未声明的任意占位符。
3. 修改前检查目标文件、权限、可用磁盘空间和备份结果。
4. 备份文件使用任务 ID 与时间戳，避免同名覆盖。
5. 回滚脚本必须在执行修改前生成并通过静态校验。
6. 安全中心记录操作链路编号、目标服务器、原状态、目标状态、执行命令摘要、备份、验证结果和回滚状态。
7. 网络、SSH、防火墙、服务停止和定时任务修改标记为高风险，并要求二次确认。
8. 日志和界面默认遮蔽密码、令牌、私钥与敏感 Header。

## 8. 界面设计

1. 快捷命令按系统、存储、网络、安全、服务和容器分组。
2. 支持名称、用途和标签搜索，并可按只读、可回滚和风险等级筛选。
3. 卡片只显示名称、简短用途、风险标识和类别，详细说明进入表单或确认页后显示。
4. 服务、网卡、容器、时区等对象使用自动扫描后的下拉框或多选框，提供“重新检测”。
5. 修改表单明确展示修改内容、备份位置、验证方法、自动回滚等待时间和快捷回滚入口。
6. 1366×768 下减少列数并纵向滚动；1920×1080 下使用更宽网格；不依赖横向滚动才能看到命令。
7. 日间和夜间主题均保证风险标签、错误提示、禁用态和辅助文字具备可读对比度。

## 9. 错误处理

- 工具缺失：使用降级命令；没有可靠降级时给出安装建议。
- 权限不足：停止执行并说明需要的权限，不自动提权猜测密码。
- 磁盘不足或备份失败：禁止执行修改。
- SSH 中断：任务转为“连接中断”，不保持“执行中”。
- 验证失败：保留结果并提供快捷回滚。
- 回滚失败：显示错误编号，保留备份与回滚脚本位置，并提供复制诊断信息。
- 非 systemd、非 Docker 或不受支持的发行版：显示能力差异，不让整个快捷命令模块崩溃。

## 10. 测试与验收

### 10.1 单元测试

- 命令目录聚合顺序、ID 唯一和旧 ID 稳定。
- 各类参数合法、边界和命令注入用例。
- Debian/Ubuntu、CentOS/Rocky/AlmaLinux 命令选择。
- 工具缺失时的降级路径。
- 修改命令必须包含备份、验证和回滚元数据。
- 复合和多行命令完成后安全中心运行计数归零。

### 10.2 组件测试

- 自动识别结果可单选、多选和重新检测。
- 风险提示、确认、取消和快捷回滚按钮状态。
- 搜索、分类与风险筛选。
- 1366×768、1920×1080、日间和夜间主题布局。

### 10.3 端到端测试

- 连续执行多个只读快捷命令，不互相阻塞。
- 通过隔离测试环境验证修改、备份、验证、保留和回滚完整流程。
- 真实测试服务器仅执行只读命令和修改命令预览，不执行网络、防火墙、SSH 等破坏性变更。
- 回归 SSH 交互、Ctrl+C、SFTP、AI、安全中心和更新入口。

### 10.4 完成标准

1. 新增功能全部可从快捷命令界面发现并正常使用。
2. 所有修改类功能均有可用备份和按钮式回滚记录。
3. 快捷命令执行结束后任务状态及时完成，可立即执行下一条。
4. 全量单元测试、关键端到端测试和本地打包冒烟通过。
5. 不降低终端首次可用时间，不增加普通 SSH 会话的常驻服务器开销。

本轮先完成本地构建与验收，不自动发布在线更新。发布前单独确认版本号、变更说明和资产名称，并按 `[新增]`、`[修复]`、`[改动]` 分类生成更新日志。
