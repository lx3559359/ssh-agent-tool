# ShellPilot 客户端体验与启动性能设计

## 目标

在不改动 SSH/SFTP 核心会话链路和现有配置数据结构的前提下，完成 AI 未读提示、API 配置精简、中文文案补齐、错误页脱敏以及主包按需加载。

## 交互设计

### AI 对话

用户位于消息底部时继续自动跟随新内容。用户向上阅读历史后停止自动滚动；新消息只按消息条目计数，同一流式回复只计一次。历史区右下角显示“回到最新”按钮和最多 `99+` 的未读数，点击或手动滚到底部后清零。

### API 配置

默认首屏仅保留 API 地址、API 密钥、模型和保存按钮。API 路径与地址放在同一基础字段中兼容原数据；模型字段继续支持拉取模型。多配置管理、服务商模板、配置名称、认证 Header、角色、语言、Skill、MCP、代理、测试连接、历史与缓存放入“高级选项”折叠区，折叠不影响保存与校验。

### 中文化

快捷命令中的 Delay、Label、Templates 使用 ShellPilot 自有翻译键；文本编辑器搜索框使用“在文本中搜索…”。英文环境保留对应英文文案。

### 错误页

界面错误生成 `SP-日期-短哈希` 编号。页面只显示错误编号、通用说明、重新加载、复制诊断信息和问题反馈入口，不显示本地路径、原始堆栈或数据目录。完整错误仍写入本机工具日志。可复制诊断信息仅包含编号、版本、系统、时间和脱敏消息。

### 按需加载

沿用现有 React `lazy`/`Suspense` 模式，将 AI 配置、服务器状态工作区与弹窗、帮助中心、更新中心和后台更新检查拆成独立 chunk。终端主布局、SSH 会话和 SFTP 文件管理保持同步加载，确保基础连接体验不受影响。

## 验收

- AI 离底后出现回到最新和准确未读数，滚到底部后清零。
- API 配置默认只展开四项核心操作，高级功能仍可使用并能保存旧配置。
- 指定英文残留在中文环境中消失。
- 错误页 DOM 与复制内容均不包含用户目录或堆栈。
- 生产构建产生 AI、状态、帮助、更新独立 chunk，主入口体积较改造前约 `1.03 MB` 明显下降。
- 针对性测试、完整单测、Lint、生产构建和包冒烟检查通过。
