# ShellPilot Agent 只读快路径与人工终端直通设计

## 背景

ShellPilot v0.4.5 的 AI 接管已经具备会话级授权、风险分类、确认、取消、审计和验证能力，但真实手动测试暴露了四个相互关联的问题：

1. Agent 运行期间，输入区右下角仍显示灰色发送图标，没有明确的进行中反馈。
2. `ip addr` 等简单读取任务会先确认分析计划，再通过依赖 Shell Integration 的交互终端发送命令；Shell Integration 最多等待约 6 秒，失败后模型还可能继续查询终端状态。
3. 底层已经把 `ip addr` 正确分类为只读且无需命令确认，但上层计划门禁仍要求用户确认一次。
4. 人工键盘输入与程序代执行命令共用安全分类。合法缩写 `ip a` 未被只读白名单覆盖，被当作未知且不可回滚命令，从而弹出高风险确认。

在用户提供的真实 VPS 上，仅使用只读命令进行基准验证：新建 SSH 连接约 1.6 秒，`ip -brief address`、`ip route show`、`cat /proc/loadavg` 分别约 1.35 秒、0.61 秒、0.61 秒。服务器本身没有造成长时间等待，主要延迟来自客户端的计划确认、模型额外轮次和 Shell Integration 跟踪链路。

## 目标

- 人工在 SSH 终端中直接键入的命令完全直通，不弹 ShellPilot 风险确认。
- Agent 明确识别的只读操作无需计划确认或命令确认。
- Agent 只读命令不依赖交互终端和 Shell Integration，通过当前 SSH 会话的独立 exec 通道执行。
- 非只读命令不能借只读工具绕过风险确认、会话边界、取消和审计。
- Agent 运行期间提供明确、连续的进行中状态；完成、失败或取消后恢复发送状态。
- 用户可以清楚看到 Agent 实际执行的命令、目标、耗时和结果，但 Agent 不自动向交互终端注入命令。
- 使用隔离测试环境验证风险操作，使用用户提供的 VPS 只验证读取和性能，不修改 VPS。

## 非目标

- 不降低 Agent 修改、删除、上传、服务控制、网络变更和本机命令的确认要求。
- 不让任意 Shell 语法进入只读快路径。
- 不自动把 Agent 命令敲入当前交互终端。
- 不改变主导航或重做 AI 面板整体布局。
- 不在本次工作中调整 Agent 资源预算。

## 方案选择

### 方案 A：扩充命令白名单

只补充 `ip a` 等缩写并替换运行图标。改动较小，但无法消除 Shell Integration 等待、通用计划确认和后续未知缩写误判。

### 方案 B：只读 SSH 快路径与来源隔离（采用）

把人工输入、Agent 读取和 Agent 风险操作拆成三条执行路径。只读 Agent 命令通过独立 SSH exec 执行，人工输入原样发送，风险 Agent 操作保留现有冻结事务确认。

### 方案 C：Agent 始终操作可见终端

Agent 像用户一样向 PTY 输入命令。可见性强，但会与用户输入、密码提示、TUI、当前工作目录和 Shell Integration 状态互相干扰，无法作为可靠自动化基础。

方案 B 在速度、透明度和安全边界之间最平衡。

## 权限与确认边界

### 人工终端输入

- 键盘输入和粘贴后按 Enter 属于用户直接控制，不进入 ShellPilot 命令确认流程。
- 包括读取、修改、删除、重启等命令在内，ShellPilot 均不阻断、不弹窗、不改写。
- 当前“SSH 终端安全保护”设置不再控制人工键盘 Enter。用户界面移除该开关；旧配置键可保留读取兼容，但不再产生人工命令拦截。
- 人工命令仍由远端 Shell 自身的权限、sudo、审计和系统策略约束。

### Agent 只读操作

- 开启当前 SSH 会话的 AI 接管后，明确分类为 `allowlisted-readonly` 的调用可以直接执行。
- 不调用通用分析计划确认，不弹命令确认，也不进入高风险事务确认。
- 每次执行前后仍校验接管授权、标签页、终端 PID、SSH 主机身份指纹和取消信号。
- 只读工具只接受静态、可审计、受输出和时间边界限制的命令。

### Agent 风险操作

- 修改、删除、上传、下载到本机、服务控制、后台任务、网络/防火墙变更、本机 CLI 和 Skill 脚本继续走风险策略。
- “二次确认”的定义是：用户开启当前 SSH 会话的 AI 接管构成第一次授权；Agent 准备执行某个具体风险事务时，再弹出一次详细确认。不得在该详细确认前额外叠加一个通用计划确认弹窗。
- `confirm_agent_plan` 不再作为任何命令的通用执行许可门禁，也不再暴露给模型用于获取执行权。只读调用直接进入只读策略，风险调用直接进入冻结事务确认。
- 继续显示目标 SSH 身份、完整命令或脚本、影响对象、最坏结果、资源影响、恢复限制和验证步骤。
- 风险事务冻结后才能确认和执行，参数、会话或端点变化会使确认失效。
- 用户取消时不得发送命令；发送后的取消必须报告远端状态是已停止、部分完成还是未知。

## 只读 SSH 快路径

### 工具语义

新增专用 `run_readonly_command` Agent 工具。它只用于结构化工具无法表达的静态读取命令，例如：

- `ip addr`
- `ip route show`
- `uname -a`
- `df -h`
- `ss -lntp`

工具描述明确要求 Agent 优先使用已有结构化读取工具，其次才使用通用只读命令。状态、日志、端口和文件范围读取仍保持专用工具名称。

### 双重分类

1. 模型调用进入策略网关时进行第一次分类。
2. 绑定当前会话、补全内部参数后，在实际 SSH dispatch 前再次分类。

两次结果都必须为 `allowlisted-readonly`。任何未知、动态、管道、命令替换、后台、资源敏感、修改或禁止命令均拒绝从只读工具执行。拒绝结果返回 Agent，由 Agent解释原因或改走风险工具，系统不得自动降级到交互终端执行。

`ip a` 可以在命令分类器中补充为 `ip address` 的只读等价缩写，但人工终端直通不依赖该白名单。

### 执行通道

- 复用当前 SSH 会话 PID 对应的后端 `run-cmd`/SSH exec 通道，不新建 SSH 登录，也不向 PTY 写入字符。
- 默认超时 15 秒，最大输出 32 KiB；超限时返回截断标记。
- 每次执行使用唯一 execution ID，并通过现有 `cancel-run-cmd` 支持取消。
- 执行结果包含退出码、截断状态、采集时间、端点快照和脱敏后的有界输出。
- 现有 `read_service_status`、`read_recent_logs`、`verify_listening_port` 的命令执行也迁移到该通道；`read_file_range` 继续使用有界 SFTP 读取。
- 风险操作完成后的只读验证同样使用该 exec 通道，避免风险事务已经执行却因 Shell Integration 不可用而无法验证。

## Agent 循环与性能

纯只读任务的数据流为：

1. 用户提交问题。
2. 模型选择一个或少量只读工具。
3. 工具直接执行并返回有界证据。
4. 模型生成最终回答。

纯只读任务不得为了获得执行权调用 `confirm_agent_plan`，也不得在读取成功后无理由调用 `get_terminal_status`。当只读 exec 失败时，应直接显示明确错误和可重试建议，不进行不可控的状态轮询。

工具执行性能目标以用户提供的 VPS 为基准：

- 已建立 SSH 会话时，单条普通只读命令从 dispatch 到结果的 p95 不超过 3 秒。
- 不再出现 Shell Integration 的 4–6 秒准备等待。
- “查看网卡信息”正常只需要一次读取工具调用和一次最终模型回答。

模型首轮和最终回答耗时受所选模型及网络影响，不承诺固定总时长，但客户端不再增加计划弹窗、终端跟踪等待和多余状态查询。

## 操作可见性

Agent 不自动把命令输入到交互终端，原因是：

- 用户可能正在输入另一条命令。
- 终端可能处于 vim、top、密码提示或其他 TUI。
- 当前目录、环境变量、alias 与独立 exec 的上下文可能不同。
- 自动注入会把可靠的控制通道重新绑定到 Shell Integration。

可见性由 Agent 工具卡承担：

- 标题显示“只读执行”、命令摘要和运行状态。
- 展示目标会话、开始时间、持续时间、退出码和是否截断。
- 输出默认折叠，用户可展开查看完整有界结果。
- 提供“复制命令”。
- 提供“填入终端”作为用户主动操作：只填入正确会话的空闲命令行，不按 Enter；终端已有输入、处于 TUI 或会话已变化时禁用。
- 保留折叠后的技术 JSON，便于排障，但不再让原始 JSON 成为主要展示。

## 运行状态 UI

- Agent 任务注册成功后，输入区右下角用旋转的 Loading 图标替代发送图标。
- 图标提示文案为“Agent 正在执行”。
- 任务完成、失败、取消或注册失败后恢复发送图标。
- 运行图标不承担取消功能，避免误触；继续使用现有“一键停止”。
- 状态来源使用 Agent 任务注册表，而不是模型流式文本，确保工具执行期间也保持进行中显示。

## 错误处理

- 当前标签页、PID、SSH 身份或接管授权变化：执行前拒绝，并返回会话已变化。
- 命令不再是只读：拒绝只读工具，不自动发送。
- 超时：取消对应 exec execution ID，返回超时和远端停止确认结果。
- 输出超限：保留前 32 KiB、设置截断标记，不继续无界读取。
- SSH exec 不可用：显示失败及“复制命令/填入终端”入口，不自动切换到 PTY。
- Agent 被停止：同时取消模型请求和活动 SSH exec；迟到结果不得更新已取消任务。

## 测试与自检

### 单元测试

- `ip a`、`ip addr`、`ip address`、`ip route` 均识别为只读。
- 只读工具拒绝修改、未知、管道、命令替换、后台和资源敏感命令。
- 纯只读调用在没有 plan grant 时可执行。
- 不再存在独立的通用 plan grant；风险调用只接受与冻结事务完全匹配的一次性确认授权。
- 风险工具仍要求冻结确认且无法借只读工具绕过。
- 人工 Enter 无论命令分类为何均不创建确认弹窗。
- Agent 运行时显示 Loading 图标，所有终态恢复 Send 图标。
- exec 输出、超时、取消、端点变化和截断行为均有覆盖。

### 本地端到端测试

使用隔离的本地 SSH fixture：

- 开启接管后请求查看网卡信息，不出现计划或风险确认。
- 工具卡展示准确命令、目标、状态、耗时和输出。
- 只读命令不写入交互终端缓冲区。
- “填入终端”只填入、不执行，并在非空输入/TUI/错会话时禁用。
- 人工输入 `ip a` 和一个测试用的已知修改命令都直接到达 fixture，不弹 ShellPilot 确认。
- Agent 风险操作仍弹详细确认；取消后 fixture 不产生变更。
- 风险操作不会先弹通用计划确认再弹风险确认；一次具体风险事务只出现一个详细确认弹窗。
- 任务运行时发送位置为旋转图标，完成、失败和取消后恢复。

### 真实 VPS 只读验收

从 `F:\SSH工具开发\VPS服务器信息.txt` 在内存中读取连接信息，不打印、不写入仓库、不进入测试报告。只执行：

- `ip -brief address`
- `ip addr`
- `ip route show`
- `uname -s`
- `cat /proc/loadavg`

验收内容：退出码、输出非空、输出边界、取消能力、连续执行稳定性和 dispatch 耗时。禁止在该 VPS 上进行写文件、重启服务、安装软件、修改网络或清理目录等操作。

### 回归测试

- Agent 会话隔离、接管撤销、风险事务、取消、Skill、SFTP 和审计测试全部通过。
- AI 接管与 Skill E2E 通过。
- 主工作区视觉回归和次级界面矩阵通过。
- 全量单元测试、lint、构建、Windows 安装包启动冒烟和便携包验证通过。

## 验收标准

1. 人工输入 `ip a` 不弹窗并正常发送。
2. 人工输入其他命令也不被 ShellPilot 拦截。
3. Agent 请求“查看网卡信息”不出现计划确认或风险确认。
4. Agent 只读执行不依赖 Shell Integration，且不会在终端中自动敲命令。
5. Agent 工具卡能够清晰查看实际命令、目标、耗时、退出码和结果。
6. 运行期间发送位置显示旋转图标，终态恢复。
7. 非只读 Agent 调用仍必须通过详细风险确认，取消时零执行。
8. 用户提供的 VPS 只读验收全部通过，单条普通读取 p95 不超过 3 秒。
9. 开启接管后，具体风险事务只出现一次详细确认，不叠加通用计划确认。

## 发布与兼容性

- 目标版本为 v0.4.6。
- 不改变用户的 SSH 连接、AI 配置或 Skill 数据格式。
- 旧的 `terminalSafetyProtection` 配置值不再拦截人工键盘输入；迁移后不再在设置页展示该开关。
- 若只读快路径出现问题，可以回滚到 v0.4.5；不得以自动回退到交互终端的方式掩盖执行通道故障。
