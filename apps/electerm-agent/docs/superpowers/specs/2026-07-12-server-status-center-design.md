# ShellPilot 服务器状态中心设计

## 目标

在客户端顶部增加“服务器状态”入口，为当前已连接的 SSH 会话提供一次性、只读、可解释的服务器巡检。状态中心覆盖系统、资源、服务、平台软件、网络、防火墙、安全和容器信息，并支持复制、导出以及发送给 AI 继续分析。

## 已确认的产品决策

- 使用顶部入口和独立宽面板，不挤压终端、SFTP 或 AI 助手。
- 仅在用户打开面板或点击“刷新检测”时采集，不做后台轮询。
- 首版完全只读，不提供重启服务、修改防火墙或编辑网络等操作。
- 自动识别服务器中的平台软件及其多个关联服务。
- 支持用户添加平台识别规则，后续扩展内部或小众平台时无需改动核心探针。

## 界面结构

状态中心从上到下分为：

1. 标题栏：当前服务器完整端点、复制结果、导出报告、发送给 AI、刷新检测。
2. 状态摘要：整体状态、运行时间、CPU/负载、内存/Swap、磁盘最高使用率、服务正常与异常数量。
3. 分类标签：总览、平台与服务、资源、网络、防火墙与安全、容器、原始结果。
4. 总览主体：左侧展示平台与服务组，右侧展示网络、防火墙和需要关注的异常。
5. 状态栏：成功探针数、权限受限数、失败探针数、检测时间和“未执行修改命令”提示。

## 数据模型

每次检测生成一个内存快照：

```js
{
  version: 1,
  collectedAt: 'ISO-8601',
  endpoint: { tabId, host, port, username, title },
  overallStatus: 'healthy | warning | critical | unknown',
  summary: {},
  system: {},
  resources: {},
  services: [],
  networks: [],
  firewall: {},
  security: {},
  containers: [],
  platforms: [],
  alerts: [],
  probes: []
}
```

快照按 SSH 标签页缓存在当前进程内，不持久化服务器运行信息。切换标签页时显示对应快照，点击刷新后替换。自定义平台识别规则使用本地安全存储。

## 探针架构

探针是固定命令、解析函数和超时配置组成的注册表。用户输入不会拼接到远程命令中。

- 环境探针：发行版、内核、主机名、CPU 核数、运行时间、初始化系统及可用命令。
- 资源探针：CPU、负载、内存、Swap、磁盘、inode 和高占用进程。
- 服务探针：systemd 已安装、运行、停止和失败服务；为 OpenRC、SysV 预留识别结果。
- 网络探针：网卡、IPv4/IPv6、链路状态、默认路由、DNS、监听端口和连接统计。
- 防火墙与安全探针：firewalld、UFW、nftables、iptables、SELinux、AppArmor、登录用户和近期失败登录。
- 容器探针：Docker、Podman、Compose 项目、容器状态和端口。
- 平台识别器：服务前缀、Description、可执行路径、WorkingDirectory、FragmentPath、安装目录、容器标签和端口。

探针最多三路并发，每项有独立超时与输出上限。单项失败只影响该项，状态中心仍展示其他结果。权限不足、命令不存在、超时和解析失败必须区分显示。

## 平台与服务组识别

识别器按以下证据归组：

1. 已知平台规则，例如宝塔、GitLab、Kubernetes、Zabbix、Jenkins。
2. systemd 服务名前缀和服务依赖。
3. 共同的可执行文件、配置文件或工作目录根路径。
4. Docker Compose 项目名和容器标签。
5. 监听端口与服务进程关联。
6. 用户自定义的名称、服务匹配规则、路径规则和容器标签规则。

识别结果标记高、中、低置信度，并显示识别依据。无法可靠命名时使用“疑似平台服务组”，不冒充确定结果。未归组的基础服务进入“其他系统服务”。

## 健康判定

- systemd `failed` 服务判定为异常。
- 磁盘或 inode 超过 80% 为警告，超过 90% 为异常。
- 内存使用 `MemAvailable` 判断，避免把缓存当作不可用内存。
- 系统负载按 CPU 核数归一化。
- 防火墙未启用只标记“需关注”，不直接判定服务器异常。
- 权限不足和不支持为未知状态，不判定正常。
- 平台状态由其服务和容器中最严重状态决定。

## AI 与报告

- “复制结果”生成简洁 Markdown 摘要。
- “导出报告”支持 Markdown 和 JSON。
- “发送给 AI”打开 AI 助手，并注入有界的结构化状态上下文。
- 服务、端口、进程和原始输出均设数量与字符上限，避免 AI 上下文失控。
- 不自动访问公网 IP 查询接口，不把服务器信息发送给第三方，只有用户点击“发送给 AI”时才进入已配置模型。

## 兼容与安全

- 优先兼容 Ubuntu、Debian、CentOS、Rocky Linux 和 AlmaLinux。
- 不使用写操作，不创建远程临时文件，不重启服务，不修改配置。
- 不自动执行 sudo；权限受限时直接报告。
- 执行前识别命令能力，缺失工具使用兼容命令或显示不支持。
- 只允许已连接 SSH 会话运行，并匹配当前标签页的完整主机、端口和账号。

## 测试与验收

- 纯函数测试覆盖探针解析、阈值和平台归组。
- 使用不同发行版的固定输出样本测试兼容性。
- 覆盖 systemd 缺失、防火墙缺失、Docker 缺失、权限不足、命令超时和部分失败。
- UI 测试覆盖未连接、检测中、部分失败、刷新、切换标签页和导出/AI 操作。
- 在用户提供的真实服务器上执行只读回归，并在前后比较文件、服务、防火墙和网络配置，确认没有状态变化。
