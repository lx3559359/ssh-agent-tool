# ShellPilot 轻立体 UI、字体与双语设计规范

日期：2026-07-19

状态：设计已确认，等待用户审阅正式规范

范围：ShellPilot 客户端的非终端 UI

## 1. 目标

在不改变任何连接、终端、文件传输、AI、备份同步和设置功能的前提下，为 ShellPilot 客户端建立统一的轻立体视觉系统，并增加独立的 UI 字体选择能力和完整的简体中文/英文界面能力。

本设计解决以下问题：

- 现有界面层级偏平，页面、卡片、控件和浮层之间的空间关系不够明确。
- 设置页、右键菜单、弹窗和辅助面板的视觉语言不完全统一。
- 客户端 UI 字体与 SSH 终端字体缺少明确的产品边界。
- 部分内置界面仍存在英文、硬编码文本和中英文长度差异导致的挤压风险。
- 主题、语言、字体需要可独立选择、预览和保存，且不能互相覆盖。

## 2. 已确认的设计决策

1. 采用“B：轻立体”方向，不使用过度拟物或强烈悬浮效果。
2. 美化范围覆盖整个客户端的非终端 UI：顶部栏、侧边栏、设置中心、配置页、AI 面板、右键菜单、弹窗、通知、表单和状态栏。
3. 保留现有 UI 主题能力，并在现有主题语义层之上统一层级、阴影、高光和圆角。
4. UI 字体为全局配置，与 UI 主题独立，不跟随主题切换。
5. 提供固定的 20 项 UI 字体预设；不扫描并暴露任意系统字体，也不随客户端额外打包字体。
6. 字体列表始终显示全部预设；系统未安装的字体置灰并说明原因。
7. 字体选择后立即预览，但只有点击“应用字体”才持久化；取消、关闭或离开时恢复到已保存字体。
8. 支持简体中文和英文；所有客户端内置文案纳入同一国际化体系。
9. 终端字体、终端字号、终端 ANSI 配色和终端渲染逻辑与 UI 字体完全隔离。
10. 不论 UI 主题、配色、字体或语言如何变化，终端画布背景始终为 `#0E0F12`，并保持高对比文字。

## 3. 范围与非目标

### 3.1 范围内

- 非终端 UI 的视觉令牌与组件状态。
- 设置中心和全部配置页面的布局、间距、文本适配及视觉层级。
- 右键菜单、下拉菜单、提示气泡、通知和模态框。
- UI 字体预设、可用性检测、即时预览、应用、取消和回退。
- 简体中文/英文选择、预览、持久化和缺失翻译检查。
- 浅色主题、深色主题和现有自定义 UI 主题的兼容策略。
- 590px、820px、常规桌面宽度及系统缩放下的响应式行为。

### 3.2 范围外

- SSH 协议、连接建立、会话恢复和命令执行逻辑。
- SFTP、文件管理、AI 工具调用、备份同步和自动更新的数据流。
- xterm/终端画布的字体、字号、行高、字符宽度和 ANSI 色表。
- 对第三方网页、远程桌面、VNC、RDP 内容进行字体或主题注入。
- 下载、内嵌或自动安装字体。
- 自动把任意系统字体加入预设列表。
- 改变现有表单字段含义、默认配置、权限、安全确认或快捷键行为。

## 4. 视觉系统

### 4.1 四级层级模型

所有非终端 UI 只能使用以下四个层级，禁止组件自行发明阴影值：

| 层级 | 用途 | 视觉表达 |
| --- | --- | --- |
| L0 页面层 | 应用背景、页面底色、主工作区背景 | 无阴影，使用基础表面色 |
| L1 控件层 | 输入框、按钮、选择器、标签、侧栏按钮 | 细边框、顶部高光或轻微内阴影 |
| L2 卡片层 | 设置分组、AI 卡片、面板、工具栏 | 与页面有明确色差，低强度短阴影 |
| L3 浮层 | 右键菜单、下拉菜单、提示、模态框、通知 | 清晰边框、中等扩散阴影，可使用遮罩 |

### 4.2 视觉令牌

在现有 UI 主题令牌基础上补充或调整语义令牌。组件只消费语义令牌，不直接写固定主题颜色。

| 语义令牌 | 浅色主题建议 | 深色主题建议 | 用途 |
| --- | --- | --- | --- |
| `surfaceBase` | `#F2F6FA` | `#10161E` | L0 页面背景 |
| `surface` | `#F8FAFC` | `#151D27` | 主面板背景 |
| `surfaceElevated` | `#FFFFFF` | `#1B2531` | L2/L3 表面 |
| `surfaceInset` | `#EDF2F7` | `#101720` | 输入区、凹面区域 |
| `borderSubtle` | `#D4DEE9` | `#2B3948` | 分隔线和普通边框 |
| `borderStrong` | `#BCCADB` | `#3C4D60` | 聚焦或浮层边界 |
| `highlightTop` | `rgba(255,255,255,.88)` | `rgba(255,255,255,.06)` | 控件顶部高光 |
| `shadowControl` | `0 2px 5px rgba(28,50,78,.10)` | `0 2px 6px rgba(0,0,0,.28)` | L1 |
| `shadowCard` | `0 7px 18px rgba(30,58,95,.11)` | `0 8px 20px rgba(0,0,0,.30)` | L2 |
| `shadowOverlay` | `0 18px 40px rgba(26,44,70,.24)` | `0 20px 46px rgba(0,0,0,.48)` | L3 |
| `radiusControl` | `7px` | `7px` | 输入和按钮 |
| `radiusCard` | `10px` | `10px` | 卡片和面板 |
| `radiusOverlay` | `10px` | `10px` | 菜单和模态框 |

最终值可在实现阶段通过视觉回归测试微调，但层级关系、用途和最大强度不能改变。深色主题的高光与阴影必须独立计算，不能简单反转浅色值。

### 4.3 组件映射

- 顶部栏：L1，渐变或高光只承担边界识别，不做高反差装饰。
- 左侧主导航：导航容器 L0，导航按钮 L1；选中态通过主题主色、轻内阴影和图标颜色共同表达。
- 设置导航：容器 L0，搜索框和条目 L1；选中条目不得仅依赖颜色，应同时具备背景或边界变化。
- 设置内容分组：L2 卡片；同一页面的卡片阴影一致。
- AI 面板、文件管理器和状态栏：外框 L1/L2，内部可滚动内容保持平整，避免层级堆叠。
- 右键菜单、下拉菜单和提示：L3；菜单项本身不再叠加阴影。
- 模态框：L3；遮罩仅降低背景干扰，不改变终端内容的实际颜色配置。
- 通知：L3；成功、警告、错误由语义色表达，不使用不同阴影强度表达严重程度。
- 主要按钮：L1，主色渐变、高光和轻阴影；按下时回到基准面。
- 输入框：L1 凹面；聚焦时使用清晰焦点环，不能仅加深阴影。

### 4.4 动效

- 悬停：最多上浮 `1px`，持续 `120–180ms`。
- 按下：立即回落到基准面，持续不超过 `100ms`。
- 菜单与弹窗：仅使用透明度和 `4–6px` 的短位移，持续 `140–180ms`。
- 不使用弹性跳动、连续呼吸阴影或大范围视差。
- 系统启用减少动态效果时，取消位移，仅保留必要的透明度变化。

## 5. UI 字体系统

### 5.1 配置边界

新增的 UI 字体配置仅作用于客户端壳层和 React/HTML 组件。以下内容不得继承 UI 字体变量：

- xterm 终端画布及其辅助文本测量节点。
- 终端背景文字功能。
- 远程 Web、RDP、VNC、SPICE 和第三方嵌入内容。
- 任何依赖等宽字符测量的终端输入、选择和渲染逻辑。

UI 字体配置建议使用独立字段 `uiFontPresetId`；现有终端 `fontFamily` 和 `terminalBackgroundTextFontFamily` 保持原义，不迁移、不覆盖。

### 5.3 字体选择器

- 设置位置：设置中心 → 界面与字体 → UI 字体。
- 入口显示当前已保存字体，展开后显示搜索框和三组预设。
- 搜索匹配中文名、英文名和常用别名；搜索不区分大小写。
- 列表始终显示全部 20 项。
- 未安装字体显示为禁用态，并显示“系统未安装 / Not installed on this system”。
- 禁用项不可进入预览态，但仍可被搜索到。
- 选择可用字体后，全客户端非终端 UI 立即进入临时预览态。
- 预览区至少同时显示中文、英文、数字、路径和常用符号。
- “应用字体”持久化当前预览；“取消并恢复”、关闭设置页或退出未确认流程时恢复已保存字体。
- 预览期间发生主题或语言切换时，字体预览保持有效，但仍未持久化。
- 当前已保存字体后来被系统卸载时，客户端启动使用系统默认字体，并给出一次非阻断提示；保存值可保留，字体恢复后自动重新生效。

### 5.4 可用性检测和回退

- 使用浏览器运行时字体检测能力判断候选字体是否可用，不读取或上传系统字体清单。
- 检测失败或 API 不可用时，以“跟随系统”为安全回退，并允许客户端正常启动。
- 字体检测不得阻塞首屏、连接恢复或终端初始化。
- 字体切换导致文本测量变化时，只触发布局重排，不重建或重连终端会话。

## 6. 主题系统

### 6.1 保留现有能力

- 保留客户端现有 UI 主题列表、主题预览、自定义主题、导入和删除能力。
- 内置主题名称可以保留品牌名；编辑器中的字段名、说明、操作按钮和错误提示必须支持中英文。
- 主题切换只更新主题语义令牌，不写入 UI 字体或界面语言配置。
- UI 字体切换只更新字体变量，不重算或覆盖主题颜色。
- 语言切换只更新文案与区域格式，不改变主题或字体。

### 6.2 终端隔离

- 任何 UI 主题的终端预览和实际终端画布都以 `#0E0F12` 作为背景基线。
- 自定义 UI 主题不得通过 `main-dark`、`surface`、`terminal:background` 或相似字段覆盖实际终端画布背景。
- 终端前景文字必须保持足够对比度；默认前景建议不低于 `#D7DEE8` 的可读水平。
- 主题编辑器可以展示终端色板信息，但 UI 主题的应用动作不得修改终端字体配置。

## 7. 简体中文与英文

### 7.1 语言能力

- 设置中心提供“简体中文 / English”选择器。
- 切换语言后当前客户端界面即时预览；确认后保存，取消则恢复。
- 首次启动默认跟随系统语言；不支持的系统语言回退到简体中文。
- 用户选择优先于系统语言，升级后保持。

### 7.2 国际化覆盖

以下内置文案必须进入统一词条系统，不允许继续以组件内硬编码字符串存在：

- 顶部栏、侧边栏、状态栏和主工作区标签。
- 设置分类、字段、帮助文本、校验错误、空状态和危险操作确认。
- 右键菜单、下拉菜单、快捷键说明和工具提示。
- UI 主题编辑器字段、导入导出、预览、删除和错误提示。
- AI 面板的内置按钮、状态、工具调用展示和非模型生成提示。
- 文件管理、SFTP、历史、密钥、日志、连接信息、安全中心、更新和帮助入口。
- 通知、模态框、更新提示、恢复提示和系统错误的客户端包装文本。

服务器返回文本、命令输出、日志原文、模型生成内容和用户自定义名称不翻译。

### 7.3 文案和布局约束

- 中文与英文词条必须表达同一动作和风险，不做逐字硬译。
- 按钮不得依赖固定字符宽度；容器允许随文案增长或换行。
- 操作按钮组空间不足时整体换行，不能压缩到文字重叠。
- 表单帮助文本和错误信息使用自然换行，不以省略号隐藏关键信息。
- 仅对文件名、主机名、不可断开的技术 ID 使用省略号，并提供悬停完整内容。
- 右键菜单为快捷键保留独立列；菜单项文本不得与快捷键相互覆盖。
- 中英文切换后焦点、滚动位置、输入值和未保存状态保持不变。

## 8. 响应式与文字防挤压

### 8.1 断点行为

- 常规桌面宽度：设置导航与内容并排，配置卡片可使用双列。
- `820px` 及以下：配置卡片切为单列，按钮组允许换行，侧栏保持可用。
- `590px` 及以下：设置导航可折叠或移到顶部；标签、输入框和帮助文本纵向排列。
- 字体选择器在窄窗口内部滚动，不增加页面最小宽度。
- 模态框最大宽度受视口约束，内容区可滚动，页脚按钮可换行。

### 8.2 CSS/组件约束

- 参与弹性或网格布局的文本容器必须允许 `min-width: 0`。
- 普通说明文本使用自然换行和 `overflow-wrap: anywhere` 作为极端回退。
- 不为中英文标签设置固定宽度；需要对齐时使用合理的 `minmax()` 范围。
- 可交互控件的最小点击区域为 `32×32px`；核心操作建议达到 `36px` 高。
- 在 100%、125%、150%、175% 和 200% 缩放下不得出现横向页面溢出、控件遮挡或文字重叠。

## 9. 状态与数据流

主题、字体和语言分别维护“已保存值”和“预览值”：

1. 打开设置页时，预览值初始化为已保存值。
2. 用户选择新值时，仅更新对应维度的预览值和客户端展示。
3. 点击应用后，只持久化该维度，不覆盖另外两个维度。
4. 点击取消、关闭或离开未确认页面时，恢复该维度的已保存值。
5. 保存失败时恢复到保存前状态，显示本地化错误提示，不能影响其他配置。
6. 配置值非法或预设已不存在时使用安全默认值，并记录可诊断日志。

建议将三个维度分别建模为：

- UI 主题：`uiThemeId`
- UI 字体：`uiFontPresetId`
- UI 语言：`uiLanguage`

字段名为实现建议，可在代码审查时根据现有配置模型调整；三个字段的独立性是强制要求。

## 10. 可访问性

- 普通正文与背景对比度至少 `4.5:1`。
- 大号文本至少 `3:1`；边框、焦点和关键图形至少 `3:1`。
- 禁用态仍需可辨识，并通过说明文字表达不可用原因。
- 所有菜单、字体项、主题项、语言项和模态框支持键盘导航。
- 焦点环在浅色和深色主题均清晰可见，不能被阴影替代。
- 状态不能只靠颜色表达；选中、警告、错误和不可用必须同时有图标、文本或边界变化。

## 11. 实施边界

- 优先在统一主题令牌和共享组件层实现，不对单个页面进行无规则覆盖。
- 保留组件事件、表单提交、数据读取和业务回调；样式重构不得改变其调用时机。
- 逐类迁移设置页、菜单和弹窗，避免一次性重写功能组件。
- 不在样式任务中重构连接、终端、AI 或同步逻辑。
- 每个迁移批次必须通过现有功能测试，并补充相应的视觉/布局测试。
- 开发必须继续使用隔离分支或工作区；在本地验证通过前不发布更新。

## 12. 验收矩阵

### 12.1 单元测试

- 四级层级令牌在浅色和深色主题均能生成完整 CSS 变量。
- UI 字体预设数量固定为 20，ID 唯一且顺序稳定。
- 字体搜索支持中文、英文、别名和大小写不敏感匹配。
- 字体可用性检测正确标记可用、不可用和检测失败三种状态。
- 字体预览、应用、取消和启动回退状态转换正确。
- 主题、字体和语言任一配置更新都不会覆盖另外两项。
- 简体中文和英文词条键集合一致，不允许新增缺失词条。
- 终端背景不从 UI 主题令牌读取，终端字体不从 UI 字体配置读取。

### 12.2 组件与视觉测试

- 浅色和深色下检查 L0–L3 的层级、边框、阴影和焦点状态。
- 覆盖顶部栏、侧栏、设置导航、配置卡片、AI 面板、右键菜单、下拉菜单、通知和模态框。
- 20 项字体逐一选择；不可用字体保持禁用并显示原因。
- 简体中文和英文分别截图；长文案、错误文案和快捷键列无重叠。
- 590px、820px 和常规桌面宽度分别生成视觉快照。
- 100%、125%、150%、175% 和 200% 缩放下检查换行、溢出和可操作性。

### 12.3 端到端回归

- 新建连接、打开本地终端、SSH 连接、断线重连和关闭会话。
- SFTP、文件管理、历史、密钥、日志和设置保存。
- AI 面板打开、输入、引用终端、工具卡片和关闭。
- UI 主题切换、预览、导入和取消。
- UI 字体预览、取消、应用、重启保持和字体缺失回退。
- 语言预览、取消、应用、重启保持和升级保持。
- 右键菜单全部原有命令仍可用，快捷键行为不变。
- 所有场景确认终端背景为 `#0E0F12`，终端文字清晰，终端字体和字号未被 UI 设置改变。

### 12.4 发布门槛

- 所有现有自动化测试通过。
- 新增单元、组件和端到端测试通过。
- 浅色/深色、简中/英文、三档窗口宽度和五档缩放的视觉检查通过。
- 至少完成一轮实际 SSH 会话人工验证，确认输入、选择、复制、粘贴和 ANSI 颜色无回归。
- 本地客户端验证通过后才能进入发布流程；不得因设计完成直接发布。

## 13. 失败与降级策略

- 主题令牌缺失：使用内置默认主题的完整令牌集。
- 字体不可用或检测异常：回退“跟随系统”，提示但不阻断启动。
- 语言词条缺失：开发/测试环境报错；生产环境回退简体中文并记录键名。
- 设置持久化失败：恢复旧值，不影响终端、连接或其他设置。
- CSS 加载或组件样式异常：核心操作仍保持可见和可点击，业务功能不能依赖阴影、动画或字体加载成功。

## 14. 完成定义

本设计完成的判断标准不是“页面看起来更立体”，而是同时满足：

- 客户端非终端 UI 形成一致、克制的四级空间层级。
- 设置页、配置页、右键菜单和弹窗在浅色/深色下统一。
- 用户可以从 20 项固定预设中搜索、预览并应用 UI 字体。
- UI 主题、UI 字体和 UI 语言互相独立。
- 简体中文和英文覆盖全部内置客户端文案。
- 长文本、窄窗口和高缩放场景没有挤压、遮挡和关键内容截断。
- SSH 终端始终保持黑色背景、清晰文字和独立终端字体。
- 所有原有功能、数据流和快捷键行为不变。
