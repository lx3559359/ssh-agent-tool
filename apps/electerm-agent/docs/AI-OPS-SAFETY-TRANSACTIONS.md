# AI 运维安全事务架构

本文说明 ShellPilot 的 AI 运维诊断、安全执行、恢复点、审计记录和 SFTP 传输保护。安全事务的目标是让已识别的修改可预览、可确认、可追踪，并在恢复资料完整时提供经过验证的回滚；它不承诺所有服务器操作都能自动恢复。

## 架构

安全链路由以下部分组成：

1. **统一入口**：SSH 终端、AI/Agent、快捷命令、服务器状态和 SFTP 都使用统一安全请求模型。
2. **命令分类器**：把命令分为 `readonly`、`change`、`unknown` 和 `blocked`。只有能够证明影响范围且存在恢复提供方的 `change` 才标记为可恢复。
3. **恢复提供方**：按文件、权限、systemd、防火墙、网络、Docker 和 SFTP 副作用准备恢复资料。准备失败时不执行受保护修改。
4. **事务执行器**：负责准备、确认、执行、验证、保留、取消和回滚，并把每次状态变化持久化。
5. **恢复绑定**：用 SHA-256 指纹绑定 endpoint、操作计划和恢复产物，防止记录被修改后继续执行。
6. **端点保护**：每一步核对主机、端口、账号和当前会话能力。连接断开后可重新连接，但只有相同 endpoint 才能继续恢复。
7. **只读任务执行器**：AI 异常诊断计划只允许运行再次分类为 `readonly` 的命令，支持逐步进度、超时、取消和停止后续步骤。
8. **安全操作中心**：聚合事务和诊断任务，按“执行中”“可回滚”“历史记录”“旧版记录”展示，并提供取消、立即回滚、重试和保留修改入口。
9. **审计与脱敏**：记录命令摘要、状态、输出摘要、恢复资料和错误原因；密码、私钥、Token、Cookie 和常见密钥字段在持久化前脱敏。

安全记录只保存 endpoint 身份，不保存认证密码、私钥正文或 API Key。恢复命令和远程快照目录同样受路径、大小、节点数量和内容校验限制。

## 状态机

修改事务使用以下状态：

```text
preparing
  -> recovery-ready
  -> awaiting-confirmation
  -> executing
  -> verification-passed
  -> rollback-available
  -> kept

rollback-available -> rolling-back -> restored
任意受控阶段 -> failed 或 cancelled
```

- `preparing`：识别影响范围并准备快照、原状态或反向操作。
- `recovery-ready`：恢复资料已写入并完成结构与恢复绑定校验。
- `awaiting-confirmation`：向用户展示原命令、风险、影响范围和恢复能力。
- `executing`：在原 endpoint 执行修改。
- `verification-passed`：执行后的目标状态已验证。
- `rollback-available`：修改已完成，可选择立即回滚或保留修改。
- `kept`：用户确认保留修改，自动回滚入口关闭，审计记录保留。
- `rolling-back`：正在执行恢复动作。
- `restored`：恢复动作与回滚验证均通过。
- `failed`：准备、执行、验证或恢复失败，记录保留具体失败阶段。
- `cancelled`：确认前取消，或执行取消已安全收口；取消后若已有副作用，必须按记录提示继续回滚。

AI 只读诊断任务使用 `draft`、`awaiting-plan-confirmation`、`running-readonly`、`awaiting-change-confirmation`、`running-change`、`completed`、`failed`、`cancelled` 和 `partially-completed`。当前异常一键诊断只进入只读计划确认与 `running-readonly`，不会自动进入修改阶段。

## 支持矩阵

| 场景 | 自动识别 | 执行前恢复点 | 取消 | 自动回滚 | 结果验证 |
| --- | --- | --- | --- | --- | --- |
| 已验证的只读终端命令 | 是 | 不需要 | Ctrl+C/任务取消 | 不适用 | 记录退出状态 |
| 可静态解析的单行文件写入 | 是 | 文件快照 | 支持 | 支持 | 内容/元数据验证 |
| chmod 与可识别权限修改 | 是 | 权限与属主快照 | 支持 | 支持 | 权限/属主验证 |
| 受信 systemd 状态修改 | 是 | 原始服务状态 | 支持 | 支持 | 服务状态验证 |
| 受信防火墙、网络、Docker 修改 | 部分 | 提供方专用快照 | 支持 | 条件支持 | 提供方专用验证 |
| 快捷命令表单修改 | 是 | 与分类结果一致 | 支持 | 条件支持 | 与恢复提供方一致 |
| AI/Agent 生成的修改命令 | 是 | 与终端统一 | 支持 | 条件支持 | 与恢复提供方一致 |
| AI 异常只读诊断计划 | 是 | 不需要 | 支持 | 不适用 | 每步退出状态与限长输出 |
| SFTP 删除、重命名、权限与编辑保存 | 是 | 远程快照 | 支持 | 支持 | 强内容/元数据校验 |
| SFTP 传输：上传、覆盖、同端复制或移动 | 是 | 目标端快照 | 支持 | 支持 | 来源与目标内容身份校验 |
| SFTP 跨端传输 | 是 | 目标端快照 | 支持 | 条件支持 | 来源 endpoint 与内容再次校验 |
| SFTP 下载到本机 | 是 | 不创建远程快照 | 支持 | 不适用 | 本地传输结果校验 |
| FTP 修改、覆盖或删除 | 有风险提示 | 否 | 取决于传输能力 | 否 | 仅报告执行结果 |
| vim/nano 内部写入 | 否 | 否 | 终端 Ctrl+C | 否 | 无法知道最终副作用 |
| 动态脚本、alias、function、复杂多行命令 | 只能判为未知或阻止 | 否 | 支持终止当前命令 | 否 | 不承诺恢复 |

“条件支持”表示必须同时满足命令可静态解析、恢复提供方可用、恢复资料通过绑定校验、当前 endpoint 匹配、目标未被外部修改以及回滚验证通过。缺少任一条件都会拒绝自动回滚或明确标记失败。

## 不可恢复边界

- **FTP 无自动回滚**：FTP 不提供与 SFTP 安全适配器等价的快照、强身份校验和恢复能力。覆盖或删除前必须自行备份。
- **交互编辑器**：`vim`、`nano`、交互 shell 和 TUI 程序内部何时写入、写入哪些文件无法从启动命令可靠判断，因此不能可靠快照。
- **动态执行**：脚本解释器、命令替换、动态变量、未知二进制、alias、function 和复杂多行命令无法证明完整影响范围。
- **业务系统内部副作用**：数据库事务、消息队列、第三方 API、云资源和应用内部写入不在文件或系统服务恢复提供方的控制范围内。
- **外部并发修改**：恢复前如果目标内容、权限或 endpoint 与执行完成时不同，系统会拒绝覆盖外部修改，要求人工处理。
- **不可逆系统操作**：格式化磁盘、重启/关机、数据库销毁和写入设备等明确危险命令会被阻止，不提供“继续执行并自动恢复”的选项。
- **断连并不等于已回滚**：网络修改可能使 SSH 立即断开。自动等待或重连只能帮助恢复流程继续，不能证明服务器已经恢复。
- **恢复失败**：权限变化、磁盘满、恢复资料损坏、服务环境变化或人工删除快照都可能使回滚失败。生产环境必须保留云控制台、VNC、业务备份和独立快照。

## 性能策略

- 普通 SSH 无额外后台开销：未触发受保护事务时不自动轮询，不启动扫描、快照、清单或结果校验任务，也不部署驻留 Agent。
- 服务器状态、平台识别和异常诊断均按需执行；只读探针限制并发数、单步超时和最大输出字节，任务取消后停止后续步骤。
- 受保护修改会产生恢复点快照、影响清单和执行后校验 I/O；快照限制路径范围、目录深度、节点数量、实际流式字节和清单大小，防止大目录拖慢服务器。
- 受保护的大文件或大目录传输会执行流式摘要、目标验证，并可能读写目标端备份，因此会增加耗时、磁盘 I/O、网络 I/O 和备份空间占用；传输不把大文件一次性加载到渲染进程内存。
- 上述保护工作仅在对应操作触发时按需运行，支持取消，并受并发、超时、输出、路径、节点、字节和清单预算限制。
- 事务记录只保存脱敏摘要和恢复元数据；大输出、完整日志和二进制内容不直接写入审计数据库。
- endpoint 校验和恢复绑定校验只在关键状态转换时执行，不影响普通只读终端输入。
- 后台监控仅跟踪当前运行任务，任务结束后释放注册表、AbortController、传输句柄和 UI 订阅。

## 故障恢复

1. 在安全操作中心找到记录，先确认来源、主机、端口、账号、命令摘要和失败阶段。
2. 如果状态是“执行中”，先尝试“取消任务”，等待执行器确认当前命令或传输已经停止。
3. 如果状态是“可回滚”，保持原 SSH 会话并点击“立即回滚”。系统会再次校验 endpoint、恢复绑定和目标当前状态。
4. 如果会话已断开，重新连接到相同主机、端口和账号。endpoint 不匹配时不得借用其他会话恢复。
5. 回滚完成后必须看到 `restored`，并确认回滚验证通过；仅看到命令退出不代表恢复成功。
6. 如果恢复失败，不要重复执行原修改。保留记录和远程恢复目录，使用云控制台或带外通道检查磁盘、权限和网络。
7. 若目标已经被外部修改，先导出审计记录并人工合并，避免自动回滚覆盖新的有效配置。
8. 旧版记录缺少完整恢复绑定，只能按兼容流程处理，不应视为与现代安全事务等价。

## 测试方法

### 本地模型测试

```powershell
node build/bin/smoke-safety-transactions.js
node --test test/unit-ci/safety-release-matrix.spec.js
npm run test-unit-ci
```

没有真实服务器凭据时，smoke 只验证分类、状态机、endpoint guard、恢复绑定和非可恢复警告，然后以结构化 JSON 输出结果并安全退出。

### 可配置真实服务器 smoke

只有显式设置 `SHELLPILOT_SAFETY_SMOKE_REAL=1`，并提供完整 SSH 环境变量时才运行远程测试。远程测试复用 SSH/SFTP smoke 的安全路径生成器，只允许在 `/tmp`、`/var/tmp` 或它们的子目录下创建随机 `shellpilot-smoke-*` 临时目录，验证临时文件写入、chmod、恢复和取消；`finally` 会校验路径后清理。它不会修改网络、防火墙、服务、系统配置或业务目录，也不会打印密码、私钥或 API Key。

### 候选版本验证

```powershell
npm run test-unit-ci
npm run lint
npm run vite-build
git diff --check
```

测试通过只表示本地候选满足代码与安全矩阵。正式在线更新还必须单独完成 Windows 打包、隔离环境安装/升级 smoke、真实服务器回归、发布资产校验和维护者发布确认；候选准备阶段不得写入更新批准清单。
