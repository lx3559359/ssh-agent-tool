# ShellPilot 中文使用帮助

## 1. 新建和管理服务器

点击顶部“新建”，填写服务器名称、IP 或域名、端口、账号，以及密码或私钥。保存后可在左侧“服务器”中重复使用。连接信息可通过“连接信息”和“备份同步”查看或导出，敏感备份请设置加密密码。

## 2. SSH 终端

连接后可直接输入命令并回车执行，无需二次确认。终端支持 Ctrl+C 中断、复制粘贴、搜索、分屏、标签页和会话日志。AI 或快捷命令生成的修改类命令会先展示预览和风险提示。

## 3. SFTP 文件管理

打开会话上方的“SFTP”即可浏览本地和远程目录，进行上传、下载、预览、重命名等操作。可将远程文件拖到 AI 对话框，或选中文件后点击“引用文件”。

## 4. 安全备份与恢复

选中远程文件或文件夹后点击“一键备份”，备份默认保存在远程服务器的 `.shellpilot-backups` 目录。通过“安全操作中心”可以查看记录、恢复最近备份和处理可恢复删除。任何涉及网络、防火墙、SSH 配置、权限或系统服务的修改，都应先确认控制台和回滚路径可用。

## 5. AI 助手和模型 API

在顶部“模型 API”中添加服务。API 地址和 API Key 为必填项，其余项目可按服务商要求填写。点击“拉取模型”后可在 AI 助手顶部切换配置和模型。AI 支持终端输出、选中文本、SFTP 文件、本地上传和粘贴文件作为上下文。

## 6. 快捷命令

顶部“快捷命令”包含系统、磁盘、内存、进程、端口、DNS、时间、防火墙、日志、Docker、Nginx 和抓包等操作。需要参数的项目会打开表单，填写后先预览命令；修改类操作会尽量创建备份、回滚脚本或快捷回滚记录。

## 7. MCP 与 CLI

MCP 可接入监控、CMDB、知识库和其他工具服务；CLI 可调用本机允许列表中的命令行工具。请仅配置可信服务，不要在未知命令中传入密码、密钥或敏感文件。

## 8. 检查更新与更新源

打开顶部“检查更新”，可选择：

- **自动选择（推荐）**：先访问 ModelScope 国内更新源，失败后回退 GitHub。
- **ModelScope 国内源**：只使用国内源。
- **GitHub**：只使用 GitHub Releases。

更新源也可以在“设置”中修改。客户端只接受带正式审批清单的更新版本，下载完成后由用户确认重启安装。

## 9. 工具日志和故障排查

左侧“日志”用于查看客户端自身运行记录。排查问题时请保留错误提示、发生时间、操作步骤和日志。必要时导出诊断包，但分享前应检查其中是否包含 IP、账号、路径或业务信息。
