# ShellPilot 中文使用帮助

ShellPilot 是一款以 SSH 终端和 SFTP 为核心、集成 AI 助手、Agent、快捷运维命令、MCP、CLI、安全备份和在线更新的 Windows 客户端。本指南面向第一次接触 SSH 的用户，也可作为日常功能手册。

## 1. 第一次使用：5 分钟快速上手

1. 点击顶部“新建”。
2. 填写服务器名称、IP 或域名、SSH 端口、账号和认证信息。
3. 点击“测试连接”，成功后保存服务器。
4. 打开 SSH 会话，在终端输入 `uptime`、`df -h` 等只读命令验证终端。
5. 切换到 SFTP 验证文件浏览；需要辅助排障时打开右侧 AI 助手。

首次连接陌生服务器时应核对主机指纹。不要在公共电脑保存生产服务器密码，也不要将密码、私钥或 API Key 发到聊天工具中。

## 2. 新建、编辑和管理服务器

顶部“新建”用于建立正式连接配置。常用字段如下：

- **名称**：方便识别服务器，例如“生产 Web-01”。
- **主机**：服务器 IP 或域名。
- **端口**：SSH 默认通常为 `22`，请以服务器实际配置为准。
- **账号**：例如 `root`、`ubuntu` 或运维账号。
- **认证方式**：密码、私钥或 SSH Agent。

保存后可在左侧“服务器”列表重复连接。服务器支持分组管理；通过右键菜单可编辑、复制、重命名或删除配置。临时测试可使用快速连接，长期使用的服务器建议保存为正式配置。

顶部“连接信息”用于查看已保存的 IP、端口、账号和认证方式，方便迁移到其他 SSH 工具。查看密码或导出连接资料前，应确认当前屏幕和文件存储环境安全。

## 3. 认证与密钥

### 3.1 密码认证

填写账号和密码即可。生产环境建议限制密码登录，并逐步使用密钥认证。不要在多人共用电脑保存高权限账号密码。

### 3.2 私钥认证

选择本机私钥文件；如果私钥本身加密，还需要填写私钥密码。左侧“密钥”可管理密钥资料。私钥文件不应上传到公开网盘、代码仓库或 AI 服务。

### 3.3 主机指纹

首次连接会显示服务器主机指纹。指纹变化可能是服务器重装或密钥轮换，也可能存在中间人风险。无法确认原因时不要直接忽略提示。

## 4. SSH 终端

连接成功后可直接输入命令并按 Enter 执行，普通手工命令不需要二次确认。

常用操作：

- `Ctrl+C`：中断当前命令。
- `Ctrl+Shift+C` / `Ctrl+Shift+V`：复制和粘贴，具体行为可在设置中调整。
- 多标签：同时保持多个服务器或本地终端会话。
- 分屏：对比不同服务器或同时观察日志与命令结果。
- 搜索：在当前终端输出中查找关键词。
- 右键菜单：复制、粘贴、搜索及其他终端操作。
- 会话历史：查看历史会话和相关记录。

长时间运行的任务建议放在 `tmux`、`screen` 或 `nohup` 中，避免网络中断导致任务停止。AI 和快捷命令生成的修改类命令会先预览，但最终仍需要用户判断是否执行。

## 5. SFTP 文件管理

连接服务器后切换到“SFTP”，界面会显示本地目录和远程目录。

支持的常用操作包括：

- 进入目录、返回上级和刷新目录。
- 上传、下载、预览和重命名文件。
- 新建目录、删除文件或文件夹。
- 将远程文件拖到 AI 对话框进行分析。
- 选中文件后使用“引用文件”让 AI 读取。
- 对选中的远程文件或文件夹执行一键备份。

上传或覆盖前请确认目标路径、目录权限、磁盘空间和同名文件。大文件传输期间不要关闭会话；传输失败时先检查网络、权限和空间。

## 5.1 服务器状态中心

顶部“服务器状态”用于对当前已连接的 SSH 服务器执行一次只读巡检。未连接、本地终端或 SSH 会话已断开时，入口会保持禁用。

使用流程：

1. 先连接目标 SSH 服务器，再点击顶部“服务器状态”。
2. 客户端会读取系统、资源、服务、网络、防火墙、安全和容器信息，并显示扫描时间与健康摘要。
3. 在“平台与服务”中查看平台与服务自动识别结果。客户端会结合服务名称、安装路径和 Compose 项目，把同一平台软件的多个服务归组并显示识别依据。
4. 平台未被识别时，可打开“平台识别规则”，填写平台名称、服务前缀、精确服务名、路径前缀或 Compose 项目名；保存后手动刷新。
5. 需要重新扫描时点击“刷新”。客户端不会定时在后台执行命令。
6. 确认内容后，可复制 Markdown、导出 Markdown/JSON，或主动点击“发送给 AI”继续排查。

状态说明：

- **正常**：命令成功执行并取得有效结果。
- **权限受限**：当前账号不能读取对应信息，不应被当作正常状态。
- **不支持**：服务器缺少对应命令、服务管理器或容器组件。
- **超时/失败**：本次探测没有在限定时间内完成或命令执行失败，可检查连接后重试。

服务器状态中心严格使用只读命令，不使用 `sudo`，不修改服务器配置，不创建临时文件，也不访问外部网站查询公网 IP。复制、导出和发送给 AI 均需用户主动操作；发送前仍应检查报告中是否包含业务名称、内网地址或其他敏感信息。

## 6. 安全备份、恢复与回滚

### 6.1 SFTP 一键备份

选中远程文件或文件夹后点击“一键备份”。备份默认保存在远程服务器的 `.shellpilot-backups` 目录，文件夹和二进制文件也会递归处理。

### 6.2 安全操作中心

安全操作中心可用于：

- 查看最近备份和操作记录。
- 恢复最近备份。
- 处理可恢复删除记录。
- 对支持的修改操作发起快捷回滚。

### 6.3 修改类命令

涉及网络 IP、路由、防火墙、SSH、权限、系统服务和配置文件的操作会尽量提供预览、备份、自动回滚脚本或快捷回滚按钮。只读查询不需要创建回滚记录。

回滚机制不能替代正式备份。修改服务器网络、默认路由、防火墙或 SSH 端口前，应确保云控制台、VNC、KVM 或其他带外登录方式可用。

## 7. 模型 API 配置

点击顶部“模型 API”添加模型服务、OpenAI 兼容中转站或本地模型。

最简配置流程：

1. 选择服务商模板，或者直接自定义。
2. 填写 API 地址和 API 密钥，这两项为必填。
3. 点击“拉取模型”，获取当前接口支持的模型。
4. 选择默认模型并点击“测试连接”。
5. 测试成功后保存。

API 地址可填写基础地址、带 `/v1` 的地址或完整接口地址，程序会尽量自动识别路径。API 路径、模型、认证 Header、角色、语言、代理、Skill 和 MCP 均为可选高级设置。

可以保存多组 API 配置。AI 助手顶部左侧下拉选择配置名称，右侧下拉选择该配置已拉取的模型。“可用”表示最近一次连接测试成功，不代表服务商永久在线。

## 8. AI 助手与 Agent

### 8.1 对话模式

适合解释命令、分析错误、总结日志和生成排查建议。输入问题后按 Enter 发送，Shift+Enter 换行。

### 8.2 Agent 模式

适合结合终端、SFTP、Skill、MCP 和 CLI 上下文分步骤完成排查。Agent 不是固定功能列表，可以继续增加 Skill 和外部能力。

### 8.3 上下文按钮

- **引用终端**：读取当前终端输出。
- **引用选中**：只读取终端中选中的文字。
- **引用文件**：读取当前选中的 SFTP 文件。
- **生成命令**：根据上下文生成命令建议。
- **引用 MCP 配置**：把已配置 MCP 能力作为上下文。
- **引用 CLI 能力**：把本机允许使用的 CLI 能力作为上下文。

### 8.4 文件附件

AI 助手支持：

- 点击上传按钮选择本地文件。
- 从剪贴板粘贴文件。
- 将 SFTP 文件直接拖到 AI 对话框。
- 上传日志、配置、脚本和常见文本资料进行分析。

发送前应检查附件是否包含密码、Token、Cookie、私钥、客户数据或其他敏感内容。

## 9. 大日志与压缩日志

大日志不要一次全部发送给 AI。建议按以下流程处理：

1. 先说明故障发生时间、现象和服务名称。
2. 使用关键词、错误码、请求 ID 或时间范围搜索。
3. 分段读取日志，查看末尾或故障时间附近内容。
4. 内容不足时继续读取下一段。
5. 将筛选后的关键内容交给 AI 分析。

客户端支持分段读取普通大日志，并可处理 `.gz`、`.zip`、`.tar.gz` 等压缩日志。压缩包很大时建议先查看文件列表，再选择目标日志，避免读取大量无关内容。

## 10. 快捷命令

顶部“快捷命令”提供服务器维护、调试和排障入口，适合不熟悉 Linux 命令的用户。

常见分类包括：

- 系统概览、磁盘、内存、Swap 和进程资源。
- TCP/UDP 端口、出口 IP、DNS、路由和时间同步。
- 防火墙状态、开放端口和网络配置。
- systemd 服务、日志关键词、Nginx 和 Docker 排查。
- 抓包、目录占用、文件权限、证书和计划任务等进阶操作。

需要参数的功能会打开表单，可填写端口、IP、域名、服务名、目录、关键词、网卡、抓包数量和保存路径。连接服务器后会尽量自动带入当前主机、SSH 端口、活动网卡、网关和 DNS。

执行前会显示用途、进阶用法和最终命令预览。涉及修改的命令会标记风险，并尽量提供备份或回滚；只读命令可直接执行。

## 11. 端口转发

顶部“端口转发”可创建与 SSH 会话关联的转发规则：

- **本地转发**：通过本机端口访问服务器或服务器内网服务。
- **远程转发**：让远程服务器通过指定端口访问本地服务。
- **动态转发**：创建 SOCKS 代理，由应用决定访问目标。

配置前确认监听地址和端口未冲突。不需要时及时关闭转发。数据库、管理后台等敏感服务不要直接监听到公网地址。

## 12. MCP、CLI 与 Skill

- **MCP**：接入 Prometheus、CMDB、知识库、工单系统和其他外部工具服务。
- **CLI**：引用本机允许列表中的命令行工具，例如已经登录的 Codex CLI。
- **Skill**：为 Agent 增加特定排查流程、领域知识、注意事项和输出要求。

只配置可信来源，并限制可访问目录、命令和数据范围。执行外部工具前检查参数，不要向未知服务传递密码、私钥和 API Key。

## 13. 备份同步与换电脑迁移

顶部“备份同步”用于备份或导入客户端配置，可包含服务器连接、分组和部分设置。

建议流程：

1. 导出前设置强加密密码。
2. 将备份保存在可信存储位置。
3. 在新电脑安装 ShellPilot 后导入备份。
4. 抽查服务器地址、端口、账号、私钥路径和 API 配置。
5. 确认迁移成功后再删除旧备份。

不要把未加密备份上传到公开网盘、GitHub 或聊天工具。

## 14. 在线更新与更新源

点击顶部“检查更新”可查看当前版本、最新版本、更新说明、下载进度和更新源。

- **自动选择（推荐）**：优先访问 ModelScope 国内源，失败后回退 GitHub。
- **ModelScope 国内源**：只使用国内更新源。
- **GitHub**：只使用 GitHub Releases。

更新源也可在“设置”中修改并保存。只有经过发布确认的版本才会推送，开发过程中的临时构建不会自动更新。更新日志统一按 `[新增]`、`[修复]`、`[改动]` 分类。

更新前保存正在执行的命令和文件传输。下载完成后按客户端提示完成更新和重启。

## 15. 主题、设置和界面

顶部“日间/夜间”用于快速切换主题，“设置”提供完整配置，包括终端字体、字号、配色、背景、快捷键、代理和启动行为。

如果日间模式文字不清晰，可调整终端主题和对比度。修改快捷键时注意与 Windows、输入法和终端程序的快捷键冲突。界面异常时可先恢复相关设置默认值，再重启客户端验证。

## 16. 工具日志

左侧“日志”记录 ShellPilot 自身运行情况，可用于排查启动、SSH、SFTP、AI、界面和在线更新问题。

反馈问题时建议提供：

- ShellPilot 版本号。
- 问题发生时间。
- 完整操作步骤。
- 完整错误提示或截图。
- 对应时间附近的工具日志。
- 是否可以稳定复现。

分享日志和诊断信息前，应检查其中是否包含 IP、账号、本机路径、Token 或业务数据。

## 17. 常见问题

### SSH 连接失败

检查服务器地址、端口、网络、防火墙、账号、密码或私钥、主机指纹和服务器 `sshd` 状态。

### SSH 可以连接但 SFTP 不可用

检查服务器是否启用 SFTP 子系统、账号目录权限、磁盘空间和当前 SSH 会话状态。

### AI 发送后提示配置问题

打开“模型 API”，确认 API 地址和 Key，重新拉取模型并测试连接。仍失败时检查代理、认证 Header 和接口是否兼容 OpenAI 格式。

### 大日志读取不完整

使用分段读取、继续读取和关键词搜索；压缩日志先查看包内文件，再选择目标内容。

### 更新检查失败或速度慢

切换到“自动选择”或“ModelScope 国内源”后重新检查；同时保留更新错误日志。

### 修改配置后服务器失联

立即使用快捷回滚、自动回滚脚本或云控制台恢复。不要反复盲目执行网络和防火墙命令。

## 18. 安全原则

1. 默认先查询、再修改。
2. 修改前备份，执行前预览。
3. 高风险操作保留控制台和回滚方案。
4. AI 建议必须人工复核。
5. 密码、私钥、API Key 和客户数据按敏感信息处理。
6. 只接入可信 MCP、CLI、Skill 和更新源。

本帮助文档会随 ShellPilot 新功能持续更新。每次正式发布前，应同步检查客户端帮助中心、离线文档和版本更新说明。
