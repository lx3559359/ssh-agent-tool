# ShellPilot UI、AI 与配置安全韧性设计

## 目标

本轮只增强客户端韧性和可验证性，不改动 SSH/SFTP 核心协议实现，不拆分终端启动链路。

## 1. 真实界面回归

- 使用隔离的 Electron 测试配置运行关键工作区，不读取用户真实连接和凭据。
- 固定覆盖 1366x768、1920x1080，以及 100%、125%、150% 缩放。
- 日间和夜间主题分别保留截图基准，并检查页面横向溢出、聊天输入区可见性、侧栏最小宽度和长文本换行。
- 中文输入法使用 composition 事件验证，确保组合输入期间按回车不会误发送。
- 截图只覆盖 ShellPilot 自有工作区；终端画布只检查几何和可见性，避免字体渲染差异造成无意义波动。

## 2. AI 长对话与请求韧性

- 历史记录默认只挂载最近 24 条，向上加载时每次增加 24 条；现有 `content-visibility` 继续作为浏览器级虚拟化保护。
- 保留最多 100 条会话记录，模型上下文仍只取最近 20 轮且限制为 24000 字符。
- 终端、文件附件、Agent 工具结果沿用并补测现有上限，不把完整大日志或二进制内容直接发送给模型。
- AI 请求增加首包超时、总时长上限、用户取消和一次有限重试。重试只发生在尚未收到响应的瞬时网络错误，认证、额度和模型错误不重试。
- 网络断开和超时使用中文可行动提示，不暴露 API Key、完整地址参数或本地堆栈。

## 3. 分包异常保护

- AI、配置、状态中心、帮助和更新等按需模块统一经过可恢复加载边界。
- 分包 404、旧缓存和动态导入失败时，当前会话只自动刷新一次；再次失败显示“重新加载模块”，不进入白屏。
- 终端核心、Xterm 初始化和 SSH 会话组件不纳入这一轮拆分。

## 4. API 配置安全与可观测性

- API Key 在持久化前通过 Electron `safeStorage` 单独加密，配置对象只保存密文；旧版明文字段首次保存时自动迁移。
- 运行时只在需要请求和编辑配置时恢复明文，诊断、日志和导出继续脱敏。
- API 配置支持 JSON 导入和导出，默认且当前版本始终排除 API Key。
- 每个提供商配置继续维护独立模型列表和高级参数，切换配置不会互相覆盖。
- 模型状态展示最近响应延迟、检测时间和失败原因；失败原因经过密钥与地址参数脱敏。

## 验收

- 新增单元测试覆盖分页、请求重试/超时、分包恢复、凭据迁移、导入导出和健康状态延迟。
- 运行现有完整单测、Lint、生产构建、打包烟测。
- 运行隔离 Electron 的关键工作区视觉矩阵并生成截图基准。
